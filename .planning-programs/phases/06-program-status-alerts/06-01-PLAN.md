```typescript
export const ALERT_THRESHOLDS = {
  instructor: { warning: 45, critical: 30 },
  hotel: { warning: 90, critical: 60 },

  materials_shipped: { warning: 10, critical: 5 },
  av_shipped: { warning: 10, critical: 5 },
  registrations: { warning: 45, critical: 30 },
  payment: { warning: 0, critical: -14 }, // 0 = at due date, -14 = 14 days past
} as const;
```

2. **Types**:
```typescript
export type AlertType = keyof typeof ALERT_THRESHOLDS;

export interface ProgramAlert {
  id: string;
  type: AlertType;
  severity: 'warning' | 'critical';
  message: string;
}

export interface AlertSummary {
  warningCount: number;
  criticalCount: number;
  alerts: ProgramAlert[];
}

export interface LogisticsReadiness {
  completed: number;
  total: number;
  warnings: number;
}
```

3. **calculateProgramAlerts function**:
- Takes `program: ProgramDetail`, `logistics: ProgramLogistics`, `unpaidRegistrations: { payment_due_date: string | null }[]`
- Returns `AlertSummary`
- Early return empty for on-demand format
- Early return empty for completed programs (days_until_start < 0)
- Check each logistics item against threshold:
  - No instructor confirmed AND daysUntil <= threshold
  - Hotels/venue/room_block not secured (in-person only)
  - BEO not final
  - Materials not sent/printed/shipped
  - AV not shipped (in-person only)
- Registration alert if < 6 registrations AND within threshold
- Payment alerts: count unpaid past due, roll up to single alert "N unpaid invoices"
- Use `<=` comparisons per requirements (not `<`)

4. **calculateLogisticsReadiness function**:
- Takes `format: string | null`, `logistics: ProgramLogistics`, `alerts: AlertSummary`
- Returns `LogisticsReadiness`
- In-person total = 10, virtual total = 6
- Count completed items based on _at timestamps or boolean flags
- In-person items: instructor, my_hotel, instructor_hotel, room_block, venue, beo (final), materials_sent, materials_printed, materials_shipped, av_shipped
- Virtual items: instructor, platform_ready, calendar_invites_sent, reminder_emails_sent, materials_sent, materials_shipped
- Warnings = count of logistics-related alerts (not registration/payment alerts)
