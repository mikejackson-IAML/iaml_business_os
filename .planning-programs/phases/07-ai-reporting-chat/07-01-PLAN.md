1. **SCHEMA_CONTEXT constant** - Minimal schema for Claude's system prompt:
   ```typescript
   export const SCHEMA_CONTEXT = `
   Available tables:
   1. program_dashboard_summary - Programs with enrollment and readiness
      Columns: id, instance_name, program_name, city, state, start_date, end_date,
               current_enrolled, min_capacity, max_capacity, readiness_score, status,
               format (in-person/virtual/on-demand), days_until_start, venue_name

   2. registration_dashboard_summary - Individual registrations with payment
      Columns: id, full_name, email, company_name, registration_date, payment_status,
               final_price, program_instance_id, program_name, city, attendance_type

   Common query patterns:
   - Filter by city: eq('city', 'Austin')
   - Filter by program name containing: ilike('program_name', '%ERL%')
   - Filter by year: gte('start_date', '2025-01-01').lt('start_date', '2026-01-01')
   - Get archived: lt('days_until_start', 0)
   - Get upcoming: gte('days_until_start', 0)
   `;
   ```

2. **QueryResult type** - Structure for query results:
   ```typescript
   export interface QueryResult {
     data: Record<string, unknown>[];
     format: 'table' | 'chart' | 'text';
     chartConfig?: {
       type: 'bar';
       xKey: string;
       yKey: string;
       title?: string;
     };
   }
   ```

3. **executeQuery function** - Takes structured query params and executes against Supabase:
   - Accept table name, filters, select columns, aggregation options
   - Build query using Supabase client methods (NO raw SQL)
   - Return typed results
   - Handle errors gracefully

4. **formatQueryResult function** - Determines display format:
   - Comparison queries (vs, compare) → chart format
   - List queries (who, which, list) → table format
   - Single value queries (how many, total) → text format

Create SSE streaming endpoint following Planning Studio pattern:

1. **Request body interface:**
   ```typescript
   interface ChatRequest {
     messages: { role: 'user' | 'assistant'; content: string }[];
     programId?: string; // Optional context for program detail page
   }
   ```

2. **System prompt** that includes:
   - SCHEMA_CONTEXT from programs-chat.ts
   - Instructions for Claude to analyze user query and respond with:
     - Natural language explanation
     - Structured data (JSON block) when query requires data lookup
   - Format selection rules (table for lists, chart for comparisons, text for counts)
   - If programId provided, note "Current program context: {name}" for context-aware queries

3. **Claude tool definition** for `query_programs`:
   ```typescript
   const tools = [{
     name: 'query_programs',
     description: 'Query program or registration data',
     input_schema: {
       type: 'object',
       properties: {
         table: { type: 'string', enum: ['program_dashboard_summary', 'registration_dashboard_summary'] },
         select: { type: 'array', items: { type: 'string' } },
         filters: { type: 'object' },
         groupBy: { type: 'array', items: { type: 'string' } },
         aggregate: { type: 'object', properties: { field: { type: 'string' }, operation: { type: 'string', enum: ['count', 'sum', 'avg'] } } },
         orderBy: { type: 'object', properties: { column: { type: 'string' }, ascending: { type: 'boolean' } } },
         limit: { type: 'number' }
       },
       required: ['table']
     }
   }];
   ```

4. **SSE streaming implementation:**
   - Create ReadableStream with TextEncoder
   - Stream text deltas as `data: {"type": "text", "content": "..."}`
   - When tool use detected, execute query and stream result as `data: {"type": "data", "result": {...}}`
   - End with `data: {"type": "done"}`
   - Handle errors with `data: {"type": "error", "message": "..."}`

5. **Response headers:**
   ```typescript
   return new Response(stream, {
     headers: {
       'Content-Type': 'text/event-stream',
       'Cache-Control': 'no-cache, no-transform',
       'Connection': 'keep-alive',
     },
   });
   ```

Use claude-sonnet-4-20250514 model. Log API usage with logApiUsage from @/lib/api/usage-tracking.
