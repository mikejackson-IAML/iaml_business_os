  <action>
Create a single comprehensive migration file following the established pattern from n8n_brain_schema.sql and dev_projects_schema.sql.

Include in this order:
1. **Header comment** with schema name, purpose, date
2. **Enable extensions:**
   - `CREATE EXTENSION IF NOT EXISTS vector;` (no schema qualifier - Supabase expects default location)
   - `CREATE EXTENSION IF NOT EXISTS pg_trgm;` (for text search fallback)
3. **Create schema:** `CREATE SCHEMA IF NOT EXISTS planning_studio;`
4. **Create tables** (in dependency order):
   - projects (no FKs)
   - user_goals (no FKs)
   - config (no FKs)
   - phases (FK to projects)
   - conversations (FK to projects, phases)
   - messages (FK to conversations)
   - research (FK to projects, phases, conversations)
   - documents (FK to projects)
   - memories (FK to projects, phases, conversations) with `embedding VECTOR(1536)`
5. **Create indexes:**
   - HNSW index on memories.embedding: `CREATE INDEX memories_embedding_idx ON planning_studio.memories USING hnsw (embedding vector_cosine_ops);` (safe on empty table)
   - Standard indexes on foreign keys and commonly queried columns (status, current_phase, etc.)
6. **Create functions:**
   - `update_updated_at()` trigger function (reusable)
   - `search_memories(query_embedding, match_count, filter_project_id, filter_memory_type)` - semantic search
   - `get_phase_context(project_id, phase_type)` - get conversation context for Claude
7. **Create triggers:** updated_at triggers for all tables with updated_at column
8. **Add comments:** COMMENT ON statements for all tables explaining their purpose

Use exact column definitions from REQUIREMENTS.md. VARCHAR constraints must match (e.g., title VARCHAR(255), status VARCHAR(50)).

Do NOT use schema qualifier for pgvector extension creation (causes issues in Supabase).
