  <action>
    Create `dashboard/src/app/api/planning/chat/route.ts` modeled on the mobile chat route pattern:

    - `export const runtime = 'nodejs';` and `export const dynamic = 'force-dynamic';`
    - POST handler accepts JSON body: `{ projectId: string, phaseType: string, conversationId?: string, message: string }`
    - No API key auth needed (this is a dashboard route, session-based).
    - Flow:
      1. Parse and validate request body (projectId, phaseType, message required).
      2. Load project via `getPlanningProject(projectId)`. Return 404 if not found.
      3. Get or find phase via `getPhaseByType(projectId, phaseType)`. If no phase found, return 400.
      4. If no conversationId provided, create a new conversation via `createConversation(projectId, phaseId)`. Return the new conversationId in the SSE stream as a special event.
      5. Save the user's message via `saveMessage(conversationId, 'user', message)`.
      6. Load context via `loadChatContext(projectId, phaseType)`.
      7. Build system message via `buildContextBlock(...)` + `getSystemPrompt(phaseType)`.
      8. Fetch existing messages for this conversation via `getConversationMessages(conversationId)` and format as Anthropic MessageParam array.
      9. Create Anthropic client: `new Anthropic()` (uses ANTHROPIC_API_KEY env var).
      10. Call `anthropic.messages.stream({ model: 'claude-sonnet-4-20250514', max_tokens: 4096, system: systemMessage, messages: formattedMessages })`.
      11. Stream response via ReadableStream + TextEncoder, sending SSE events:
          - First event if new conversation: `data: {"type":"conversation_created","conversationId":"..."}\n\n`
          - Text deltas: `data: {"type":"text","content":"..."}\n\n`
          - Done: `data: {"type":"done"}\n\n`
          - Error: `data: {"type":"error","message":"..."}\n\n`
      12. After stream completes, save the full assistant message via `saveMessage(conversationId, 'assistant', fullContent)` where fullContent is accumulated from stream deltas.
    - Return Response with Content-Type: text/event-stream headers (same as mobile chat route).
    - Wrap everything in try/catch. On error, send error SSE event and close stream.
  </action>
