Create `dashboard/src/lib/planning/phase-transitions.ts` with:

1. **Marker constants:**
   - `PHASE_COMPLETE_MARKER = '<!--PHASE_COMPLETE-->'`
   - `READINESS_PASS_MARKER = '<!--READINESS_PASS-->'`
   - `READINESS_FAIL_MARKER_PREFIX = '<!--READINESS_FAIL:'` (ends with `-->`; reason is between `:` and `-->`)

2. **Detection functions:**
   - `detectCompletionMarker(content: string): boolean` -- checks if PHASE_COMPLETE_MARKER exists in content
   - `detectReadinessMarker(content: string): { passed: boolean; reason?: string } | null` -- returns null if no marker, { passed: true } or { passed: false, reason } if found
   - `stripMarkers(content: string): string` -- removes all markers from display content (replace with empty string, trim extra whitespace)

3. **Incubation durations config:**
   ```ts
   export const INCUBATION_DURATIONS: Record<PhaseType, number> = {
     capture: 24,    // hours
     discover: 36,
     define: 0,      // no incubation
     develop: 24,
     validate: 0,
     package: 0,
   };
   ```

4. **Transitions requiring readiness checks:**
   ```ts
   export const READINESS_CHECK_TRANSITIONS: Array<{ from: PhaseType; to: PhaseType }> = [
     { from: 'discover', to: 'define' },
     { from: 'develop', to: 'validate' },
   ];
   ```
   - `requiresReadinessCheck(currentPhase: PhaseType): boolean`

5. **Core transition function:**
   `completePhase(projectId: string, phaseType: PhaseType): Promise<{ success: boolean; error?: string; nextPhase?: PhaseType; incubationEndsAt?: string }>`
   - Uses `createServerClient()` from `@/lib/supabase/server`
   - Updates phase record: `status = 'complete'`, `completed_at = now`
   - If incubation duration > 0: sets `incubation_ends_at` on phase record AND `phase_locked_until` on project (now + duration hours)
   - Advances `project.current_phase` to next phase in PHASE_ORDER
   - If next phase record doesn't exist, creates it with `status = 'in_progress'`, `started_at = now`
   - If next phase record exists, updates `status = 'in_progress'`, `started_at = now`
   - Returns nextPhase and incubationEndsAt if applicable

6. **Skip incubation function:**
   `skipIncubation(projectId: string): Promise<{ success: boolean; error?: string }>`
   - Sets `project.phase_locked_until = null`, `project.incubation_skipped = true`
   - Clears `incubation_ends_at` on the current incubating phase record

7. **Navigate to phase function:**
   `navigateToPhase(projectId: string, targetPhase: PhaseType): Promise<{ success: boolean; error?: string }>`
   - Updates `project.current_phase` to targetPhase
   - If target phase has no record, creates one with `status = 'in_progress'`
   - Does NOT re-incubate on revisits
   - If navigating forward past incomplete phases, just sets current_phase (UI will warn)

8. **Ensure all phases exist:**
   `ensureAllPhasesExist(projectId: string): Promise<void>`
   - Queries existing phase records for project
   - For each PhaseType in PHASE_ORDER not found, inserts with `status = 'not_started'`
