    1. Create POST `/api/planning/research/route.ts`:
       - Accepts: `{ projectId, conversationId, phaseId, query }` from request body
       - Creates a research record in `planning_studio.research` with status 'pending'
       - Updates status to 'running'
       - Calls Perplexity API synchronously (await the response — per RESEARCH.md recommendation, avoid fire-and-forget serverless issues):
         ```
         POST https://api.perplexity.ai/chat/completions
         Authorization: Bearer ${process.env.PERPLEXITY_API_KEY}
         Body: { model: 'sonar-pro', messages: [system + user query], return_citations: true }
         ```
       - On success: update record with status 'complete', raw_results (full API response), summary (choices[0].message.content), key_findings ({ citations, model }), completed_at
       - On failure: update record with status 'failed', raw_results ({ error: message })
       - Return the completed/failed research record as JSON
       - Use createServerClient pattern matching existing API routes (check chat/route.ts for import pattern)

    2. Also add GET handler to same route for fetching research by id:
       - Query param: `?id=uuid` returns single research record
       - Query param: `?projectId=uuid` returns all research for project, ordered by created_at desc
       - Query param: `?conversationId=uuid` returns research for conversation

    3. Update system-prompts.ts:
       - Add research marker instructions to the system prompt, similar to how doc generation markers are instructed. Add to each phase's prompt (or the shared base):
         ```
         When the user's question would benefit from external research (market data, competitor analysis, industry trends, technical comparisons, etc.), suggest research by including a marker:
         <!-- RESEARCH: the specific research query here -->
         Include the marker inline in your response. Be specific in the query — it will be sent to a research API.
         ```
       - Place this instruction near the doc generation marker instructions for consistency.

    Soft limits: Check count of research records for the conversation (session limit: 10) and project (project limit: 50). If at limit, return 429 with a warning message. These are soft — the UI will show a warning but the check is server-side.
