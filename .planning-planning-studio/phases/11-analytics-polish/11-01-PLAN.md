Add two new query functions to planning-queries.ts:

1. `getAnalyticsMetrics(period: AnalyticsPeriod)` - Returns:
   - `shippedCount`: number of projects with shipped_at in period
   - `avgVelocityDays`: average (shipped_at - created_at) in days, rounded to 1 decimal
   - `capturedCount`: number of projects created in period
   - `trendData`: array of { date: string, shipped: number, captured: number } for sparklines (daily buckets for week, weekly for month/quarter, monthly for all)

Period boundaries:
- week: last 7 days
- month: last 30 days
- quarter: last 90 days
- all: no filter

2. `getFunnelData()` - Returns array of { name: string, value: number } for each status:
   - Idea, Planning, Ready to Build, Building, Shipped
   - Exclude archived

Use existing supabase.schema('planning_studio') pattern. Calculate velocity only for projects with shipped_at NOT NULL.

Export type `AnalyticsPeriod = 'week' | 'month' | 'quarter' | 'all'`.
