# Phase 02, Plan 02: Search and Filter

## Objective

Add search and filter bar above the pipeline board so users can quickly find projects by title or narrow the view by status/phase.

Purpose: With many projects, the board needs filtering to stay usable. Search + filter provide zero-friction navigation.

Output: Search bar with real-time filtering and status/phase filter dropdowns integrated into the pipeline board.

## Context

@dashboard/src/app/dashboard/planning/components/pipeline-board.tsx
@dashboard/src/dashboard-kit/types/departments/planning.ts

Create `pipeline-search-filter.tsx`:
- 'use client'
- Props: `{ searchQuery: string; onSearchChange: (query: string) => void; statusFilter: ProjectStatus | 'all'; onStatusFilterChange: (status: ProjectStatus | 'all') => void; phaseFilter: PhaseType | 'all'; onPhaseFilterChange: (phase: PhaseType | 'all') => void; }`
- Layout: `flex items-center gap-3 flex-wrap`
- Search input:
  - Use shadcn `Input` component
  - Search icon (lucide `Search`) as left adornment (position absolute inside a relative wrapper)
  - `placeholder="Search projects..."`
  - `className="pl-9 w-64"` to leave room for icon
  - Controlled value from `searchQuery` prop, calls `onSearchChange` on change
- Status filter:
  - Use shadcn `Select` component (SelectTrigger, SelectContent, SelectItem)
  - Options: "All Statuses", then each non-archived status with `getStatusLabel()`
  - Value controlled by `statusFilter` prop
- Phase filter:
  - Use shadcn `Select` component
  - Options: "All Phases", then each phase with `getPhaseLabel()`
  - Value controlled by `phaseFilter` prop
- If any filter is active (searchQuery or status !== 'all' or phase !== 'all'), show a small "Clear" button that resets all filters
  </action>

Update `pipeline-board.tsx` to add filtering:
1. Add state: `searchQuery` (string), `statusFilter` (ProjectStatus | 'all'), `phaseFilter` (PhaseType | 'all') — all default to empty string / 'all'
2. Render `<PipelineSearchFilter>` above the columns container, passing state and setters as props
3. Compute filtered data:
   - Start with `projectsByStatus` local state (from Plan 01)
   - If `searchQuery` is non-empty: filter each column's projects where `title.toLowerCase().includes(query)` OR `one_liner?.toLowerCase().includes(query)`
   - If `statusFilter !== 'all'`: only show the matching column (hide others)
   - If `phaseFilter !== 'all'`: filter projects where `current_phase === phaseFilter`
4. Pass filtered projects to each `PipelineColumn`
5. Update column count badges to reflect filtered count
