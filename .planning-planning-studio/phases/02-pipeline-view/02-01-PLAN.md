# Phase 02, Plan 01: Pipeline Board with Drag-and-Drop

## Objective

Build the core Kanban pipeline board showing all projects organized by status columns with drag-and-drop to change status. This replaces the current "coming soon" placeholder with a fully functional pipeline view.

Purpose: This is the main dashboard view users interact with daily -- the central hub for seeing all project statuses at a glance and moving projects through the pipeline.

Output: Working Kanban board with draggable cards, droppable columns, optimistic status updates, and navigation to project detail.

     - Last activity: `<p className="text-xs text-muted-foreground mt-2">{formatRelativeTime(project.updated_at)}</p>` — create a simple `formatRelativeTime` helper inline or at top of file that shows "2h ago", "3d ago", etc.
   - Incubation state: detect via `project.phase_locked_until && new Date(project.phase_locked_until) > new Date()`. If incubating:
     - Add `opacity-60` to the Card
     - Show a Lock icon (from lucide-react) next to title
     - Show countdown badge: `<Badge variant="secondary" className="text-xs">{getIncubationTimeRemaining(project as any)}</Badge>` — note: `getIncubationTimeRemaining` expects `PlanningProject` but `PlanningProjectSummary` has `phase_locked_until` which is sufficient. Cast or extract the field.
   - isDragging state: `opacity-50` when dragging
   - Wrap entire card in `<Link href={/dashboard/planning/${project.id}}>` — but this conflicts with drag. Instead: use `onClick` handler that navigates via `useRouter().push()` ONLY if the drag didn't happen. Track with a ref: set `isDraggingRef` true on drag start, check on click. Or simpler: add a small clickable area (title link) while the card itself is the drag handle.
   - SIMPLEST approach: Make the card draggable, and add `<Link>` on the title text only. The title is clickable to navigate, the rest of the card is the drag handle. Use `e.stopPropagation()` on the Link to prevent drag interference.
