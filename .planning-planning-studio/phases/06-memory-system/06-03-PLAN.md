Create `dashboard/src/app/api/planning/memories/search/route.ts`:
- POST accepting `{ query: string, projectId?: string, limit?: number, threshold?: number }`
- Call generateEmbedding(query) to get query vector
- Call Supabase RPC: `.schema('planning_studio').rpc('search_memories', { query_embedding: JSON.stringify(queryEmbedding), match_threshold: threshold || 0.7, match_count: limit || 10, filter_project_id: projectId || null })`
- CRITICAL: JSON.stringify the embedding for pgvector serialization
- Return `{ results: Array<{ id, content, memory_type, similarity, project_id, source_phase, conversation_id }> }`
- 400 on missing query, 500 on error
- `export const runtime = 'nodejs'`

Create `dashboard/src/app/api/planning/ask/route.ts`:
- POST accepting `{ question: string, projectId?: string, conversationHistory?: Array<{ role: string, content: string }> }`
- Standard RAG flow:
  1. Embed the question via generateEmbedding
  2. Search memories via the search_memories RPC (same logic as search route, inline — don't call the search route)
  3. If no results, return `{ answer: "I don't have any memories about that yet. Have some conversations first and I'll remember the important parts.", sources: [] }`
  4. Build context from top results (max 10)
  5. Call Claude (claude-sonnet-4-20250514) with system prompt: "You are a planning assistant. Answer the user's question based on the memories from their planning sessions. Cite specific memories when relevant. If the memories don't contain enough information, say so honestly. Be conversational and helpful."
  6. Include the memory context as a formatted block in the user message: each memory as "- [{memory_type}] {content}"
  7. If conversationHistory provided, include as prior messages for multi-turn support
  8. Return `{ answer: string, sources: Array<{ id, content, memory_type, similarity }> }`
- `export const runtime = 'nodejs'`
