Install cmdk: `cd dashboard && npm install cmdk`

Create `dashboard/src/app/dashboard/planning/components/global-search-modal.tsx`:
- "use client" component
- Uses cmdk's Command component for the modal structure (Spotlight-style overlay)
- State: open boolean, messages array, input text, loading, sources
- Keyboard listener: Cmd+K (Mac) / Ctrl+K (Windows) toggles open state
- UI layout:
  - Dark overlay backdrop when open
  - Centered modal (max-w-2xl, rounded-xl, shadow-2xl)
  - Input at top with search icon and placeholder "Ask about any project..."
  - Below input: conversation messages area (scrollable)
  - For assistant messages: react-markdown rendering with source badges showing project name + memory type
  - Close on Escape or clicking backdrop
  - Small hint text at bottom: "Cmd+K to open - searches across all projects"
- On submit:
  1. Add user message to local messages
  2. POST to /api/planning/ask with { question, conversationHistory: messages } — NO projectId (cross-project search)
  3. Add assistant response with sources
  4. Sources should display project name (fetch from source data or include in API response)
- When modal closes, keep conversation state so reopening shows history
- Reset conversation with a "New conversation" button or icon

For cmdk integration: Use `<Command.Dialog>` for the modal with `<Command.Input>` for the search field. However, since we want a conversational interface (not just a command list), use cmdk primarily for the modal shell and keyboard shortcut handling. The actual content is our custom chat UI inside the dialog.

Alternatively, if cmdk's Dialog is too opinionated for a chat UI, build a custom modal with just the keyboard shortcut from cmdk or a simple useEffect keydown listener. Choose whichever produces cleaner code. The key requirement is: Cmd+K opens it, Escape closes it, and it has a conversational Ask AI interface.
