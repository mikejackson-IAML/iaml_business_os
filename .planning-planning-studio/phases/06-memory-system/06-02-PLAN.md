  <action>
Modify the existing chat route to trigger memory extraction after the assistant response is complete and saved.

After the SSE stream completes and the assistant message is saved to the database:
1. Fetch all messages for this conversation via getConversationMessages(conversationId)
2. Build a conversationText string: join all messages as "role: content" lines
3. Fire-and-forget: call extractMemories(conversationText, project.name, phaseType) — do NOT await in the response path
4. After extraction returns (async), POST to /api/planning/memories with the extracted memories, projectId, and conversationId
5. Send a final SSE event `{ type: 'memories_extracted', count: N }` if extraction succeeds — BUT since the stream may already be closed, wrap in try/catch and silently ignore errors

IMPORTANT: The extraction must NOT block the SSE stream. Use a pattern like:
```typescript
// After stream ends and message is saved
const extractionPromise = (async () => {
  try {
    const messages = await getConversationMessages(conversationId);
    const text = messages.map(m => `${m.role}: ${m.content}`).join('\n');
    const memories = await extractMemories(text, project.name, phaseType);
    if (memories.length > 0) {
      // Store via direct Supabase insert (not API call to avoid circular)
      // Insert memories + fire-and-forget embeddings
    }
  } catch (e) {
    console.error('Memory extraction failed:', e);
  }
})();
// Do NOT await extractionPromise — fire and forget
```

Actually, since the SSE response is already being sent, the extraction should run after the response completes. Use the pattern of kicking off the async work without awaiting it. The Node.js process will keep the promise alive.

Do NOT call the memories API route from within the chat route (circular). Instead, import the Supabase admin client and insert directly, plus call generateEmbedding for each memory and update with JSON.stringify(embedding).
