   - ICP: Target Market, Demographics, Psychographics, Pain Points, Goals, Decision Criteria, Channels
   - Competitive Intel: Market Overview, Direct Competitors (table), Indirect Competitors, Competitive Advantages, Market Gaps
   - Lean Canvas: Problem, Solution, Key Metrics, Unique Value Prop, Unfair Advantage, Channels, Customer Segments, Cost Structure, Revenue Streams
   - Problem Statement: Problem Description, Who's Affected, Current Solutions, Why They Fail, Impact/Urgency, Success Criteria
   - Feature Spec: Overview, User Stories, Feature Details (prioritized), Acceptance Criteria, Out of Scope
   - Technical Scope: Architecture Overview, Tech Stack, Data Model, API Surface, Integrations, Non-Functional Requirements, Risks
   - GSD PROJECT.md: Project Name, Vision, Problem, Target Users, Success Metrics, Constraints
   - GSD REQUIREMENTS.md: Functional Requirements (REQ-XXX format with priority), Non-Functional Requirements, Assumptions
   - GSD ROADMAP.md: Phase Overview table, Phase details with goals/tasks/success criteria, Dependencies, Risk Mitigation

   Each template should instruct Claude: "Fill known sections with project context. Mark unknown sections as [TBD - discuss in next session]. Cross-reference other documents where relevant."

Create doc-generation.ts following the exact pattern from memory-extraction.ts (tool_choice with Claude):

1. **detectDocGenerateMarker(content: string): string | null** — Regex match `<!--GENERATE_DOC:(\w+)-->`, return doc type or null. Same pattern as phase-transitions.ts marker detection.

2. **stripDocMarkers(content: string): string** — Remove all `<!--GENERATE_DOC:xxx-->` markers from content. Same pattern as stripPhaseMarkers in phase-transitions.ts.

3. **getLatestVersion(supabase, projectId: string, docType: string): Promise<number>** — Query planning_studio.documents for highest version of this project+docType pair. Return 0 if none exists.

4. **generateDocument(projectId: string, docType: DocumentType, projectContext: string): Promise<string>** — Call Claude API with tool_choice (like memory-extraction.ts pattern):
   - System prompt: Include the template from DOC_TEMPLATES[docType], plus instructions to fill with project context, mark gaps as [TBD]
   - User message: projectContext (conversation history, existing docs, memories)
   - Tool: `generate_document` with single `content` string property
   - tool_choice: `{ type: 'tool', name: 'generate_document' }`
   - Parse tool use result, return the content string
   - Use claude-sonnet-4-20250514 model (same as chat route)

5. **saveDocumentVersion(supabase, projectId: string, docType: DocumentType, content: string): Promise<{ id: string; version: number }>** — Get latest version, increment, insert new row into planning_studio.documents with file_path from DOC_FILE_PATHS. Return the new document's id and version.

6. **loadDocGenerationContext(supabase, projectId: string): Promise<string>** — Load full project context for generation: project details, existing documents (latest versions), recent memories, recent conversation summaries. Format as a text block. Similar to loadChatContext in system-prompts.ts but focused on document generation needs.

Import Anthropic SDK same way as memory-extraction.ts. Use createClient from @supabase/supabase-js with service role key for server-side operations (follow existing pattern).
