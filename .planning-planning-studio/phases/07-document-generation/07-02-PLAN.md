  <action>
Create POST route at /api/planning/documents/generate. Follow the pattern from existing API routes (memories/route.ts).

Request body: `{ projectId: string, docType: string }`

Logic:
1. Validate projectId and docType (docType must be one of the 9 valid types)
2. Create Supabase service client (same pattern as other routes)
3. Call `loadDocGenerationContext(supabase, projectId)` to gather project context
4. Call `generateDocument(projectId, docType, context)` to generate via Claude
5. Call `saveDocumentVersion(supabase, projectId, docType, content)` to persist
6. Return `{ id, version, docType, content }` with status 200

Error handling: try/catch, return 500 with error message on failure. Return 400 for invalid docType.

Note: This route imports from doc-generation.ts and doc-templates.ts (Plan 01). The route file can be created now with the imports — they'll resolve once Plan 01 completes. Both plans are Wave 1 so they execute in the same session or sequentially.
  </action>
  <verify>File exists and TypeScript structure is valid. Manual test after Plan 01 completes.</verify>
  <done>POST /api/planning/documents/generate accepts projectId+docType, calls Claude, saves version, returns document</done>

  <name>Task 2: Document fetch and update endpoint</name>
  <files>dashboard/src/app/api/planning/documents/[docId]/route.ts</files>
  <action>
Create GET and PUT routes at /api/planning/documents/[docId].

**GET handler:**
- Extract docId from params
- Query planning_studio.documents where id = docId
- Return document object (id, project_id, doc_type, content, version, file_path, created_at)
- Also accept optional query param `?projectId=X&docType=Y&version=all` to fetch all versions for version history dropdown — query planning_studio.documents where project_id = X and doc_type = Y, ordered by version desc, select id, version, created_at (no content for list view)
- If `?version=N` specified, fetch specific version

**PUT handler:**
- Extract docId from params
- Request body: `{ content: string }`
- Fetch existing document to get project_id and doc_type
- Call saveDocumentVersion with the new content (creates new version row)
- Return the new document `{ id, version, docType, content }`

Follow existing API route patterns for Supabase client creation and error handling.
  </action>
