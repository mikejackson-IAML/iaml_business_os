**export/route.ts:**
POST route. Request body: `{ projectId: string }`

Logic:
1. Create Supabase service client
2. Query planning_studio.documents for all latest-version documents for this project:
   - For each unique (project_id, doc_type), get the row with highest version
   - Select: id, doc_type, content, version, file_path
3. Return JSON: `{ documents: [...], projectName: string }`
   - Also fetch project name from planning_studio.projects for the ZIP filename

This route just returns data. ZIP generation happens client-side (per research recommendation — no server memory/temp files).

Error handling: 400 for missing projectId, 500 for query errors.
  </action>
