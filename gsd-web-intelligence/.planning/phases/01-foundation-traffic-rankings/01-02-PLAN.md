# Plan 01-02: Traffic Collection Workflows

## Goal
Build the three traffic collection workflows that fetch data from GA4 and store in Supabase.

## Wave
2 (Depends on 01-01 for schema)

## Prerequisites
- 01-01-PLAN.md completed (schema exists)
- GA4 credentials registered in n8n-brain
- GA4 property ID known

## Tasks

### Task 1: Build TRF-01 - Daily Traffic Collector
**n8n Workflow:** "Web Intel - Traffic - Daily Collector"

**Schedule:** Daily at 6:00 AM CT

**Nodes:**
1. Schedule Trigger (6:00 AM CT)
2. GA4 API - Fetch aggregate metrics for yesterday
   - Metrics: sessions, users, pageviews, bounceRate, avgSessionDuration
3. Transform data to match `daily_traffic` schema
4. Supabase - Upsert to `web_intel.daily_traffic`
5. Log success to `web_intel.collection_log`
6. Error Handler → Slack alert + n8n-brain error log

**GA4 Query:**
```json
{
  "dateRanges": [{"startDate": "yesterday", "endDate": "yesterday"}],
  "metrics": [
    {"name": "sessions"},
    {"name": "totalUsers"},
    {"name": "screenPageViews"},
    {"name": "bounceRate"},
    {"name": "averageSessionDuration"}
  ]
}
```

### Task 2: Build TRF-04 - Page Performance Collector
**n8n Workflow:** "Web Intel - Traffic - Page Performance"

**Schedule:** Daily at 6:15 AM CT

**Nodes:**
1. Schedule Trigger (6:15 AM CT)
2. GA4 API - Fetch per-page metrics (top 100 by pageviews)
   - Dimensions: pagePath
   - Metrics: sessions, pageviews, avgTimeOnPage, bounceRate, exitRate
3. Transform data to match `page_traffic` schema
4. Supabase - Upsert to `web_intel.page_traffic`
5. Log success
6. Error Handler

**GA4 Query:**
```json
{
  "dateRanges": [{"startDate": "yesterday", "endDate": "yesterday"}],
  "dimensions": [{"name": "pagePath"}],
  "metrics": [
    {"name": "sessions"},
    {"name": "screenPageViews"},
    {"name": "averageSessionDuration"},
    {"name": "bounceRate"}
  ],
  "limit": 100,
  "orderBys": [{"metric": {"metricName": "screenPageViews"}, "desc": true}]
}
```

### Task 3: Build TRF-05 - Traffic Source Breakdown
**n8n Workflow:** "Web Intel - Traffic - Source Breakdown"

**Schedule:** Daily at 6:30 AM CT

**Nodes:**
1. Schedule Trigger (6:30 AM CT)
2. GA4 API - Fetch by source/medium
   - Dimensions: sessionSourceMedium
   - Metrics: sessions, users, conversions
3. Transform and categorize (organic, direct, referral, social, email, paid)
4. Supabase - Store as JSONB in `daily_traffic.source_breakdown`
5. Log success
6. Error Handler

### Task 4: Register Workflows in n8n-brain
After creating each workflow:
1. Get workflow ID from n8n
2. Call `register_workflow` for each
3. Mark as `untested` initially

### Task 5: Store Pattern
After TRF-01 is working:
1. Export workflow JSON
2. Call `store_pattern` with:
   - name: "GA4 Daily Collector"
   - services: ["ga4", "supabase"]
   - node_types: ["schedule", "httpRequest", "postgres"]

## Acceptance Criteria
- [ ] TRF-01 runs and stores data in `daily_traffic`
- [ ] TRF-04 runs and stores data in `page_traffic`
- [ ] TRF-05 runs and stores source breakdown
- [ ] All three workflows registered in n8n-brain
- [ ] GA4 collector pattern stored for reuse
- [ ] Error handling works (tested by triggering error)
