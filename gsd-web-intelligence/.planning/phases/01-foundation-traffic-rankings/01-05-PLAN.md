## Tasks

### Task 1: Build RNK-02 - Rank Change Detector
**n8n Workflow:** "Web Intel - Rankings - Change Detector"

**Schedule:** Daily at 5:30 AM CT

**Nodes:**
1. Schedule Trigger (5:30 AM CT)
2. Supabase - Query today's rankings with yesterday's (using view)
3. Code Node - Categorize changes:
   - **Critical drops**: Lost >10 positions or fell off page 1
   - **Warning drops**: Lost 4-10 positions
   - **Gains**: Gained >5 positions
   - **New rankings**: Was null, now ranking
   - **Lost rankings**: Was ranking, now null
4. Supabase - Store changes in `web_intel.alerts` with category
5. IF critical or warning changes exist:
   - Trigger RNK-03 webhook with change data
6. Log success with change counts
7. Error Handler

**Change Detection Logic:**
```javascript
const changes = rankings.map(r => {
  const change = r.today_position - r.yesterday_position;

  if (r.yesterday_position && !r.today_position) {
    return { ...r, type: 'lost_ranking', severity: 'critical' };
  }
  if (!r.yesterday_position && r.today_position) {
    return { ...r, type: 'new_ranking', severity: 'info' };
  }
  if (change > 10 || (r.yesterday_position <= 10 && r.today_position > 10)) {
    return { ...r, type: 'major_drop', severity: 'critical' };
  }
  if (change > 3) {
    return { ...r, type: 'drop', severity: 'warning' };
  }
  if (change < -5) {
    return { ...r, type: 'gain', severity: 'info' };
  }
  return null;
}).filter(Boolean);
```

### Task 2: Build RNK-03 - Rank Alert Notifier
**n8n Workflow:** "Web Intel - Rankings - Alert Notifier"

**Trigger:** Webhook (from RNK-02)

**Nodes:**
1. Webhook Trigger
2. Code Node - Group alerts by severity
3. Code Node - Format Slack message:
   - Critical changes first (red emoji)
   - Warning changes second (yellow emoji)
   - Gains last (green emoji)
4. Slack - Send to #web-intel
5. Update alert records with `notified_at`
6. Log success
7. Error Handler

**Slack Message Format:**
```
📊 *Ranking Changes for {date}*

🔴 *Critical* ({count})
• "{keyword}" dropped from #{old} to #{new} ({url})
• "{keyword}" lost ranking entirely

🟡 *Warning* ({count})
• "{keyword}" dropped from #{old} to #{new}

🟢 *Gains* ({count})
• "{keyword}" improved from #{old} to #{new}

<{dataforseo_link}|View Details>
```

### Task 3: Build RNK-05 - Keyword Opportunity Finder
**n8n Workflow:** "Web Intel - Rankings - Opportunity Finder"

**Schedule:** Weekly Monday at 4:00 AM CT

**Nodes:**
1. Schedule Trigger (Monday 4:00 AM CT)
2. Supabase - Fetch rankings in "striking distance":
   - Positions 4-10 (page 1 improvement potential)
   - Positions 11-20 (page 2, could reach page 1)
3. Supabase - Join with search volume data
4. Code Node - Score opportunities:
   - Score = search_volume × (11 - position) for page 1
   - Score = search_volume × (21 - position) × 0.5 for page 2
5. Code Node - Sort by score, take top 20
6. Supabase - Store in `web_intel.alerts` as opportunity type
7. Code Node - Format opportunity report
8. Slack - Send opportunity digest to #web-intel
9. Log success
10. Error Handler

### Task 4: Build RNK-06 - New Keyword Discovery
**n8n Workflow:** "Web Intel - Rankings - Keyword Discovery"

**Schedule:** Weekly Wednesday at 4:00 AM CT

**Nodes:**
1. Schedule Trigger (Wednesday 4:00 AM CT)
2. Supabase - Get our top-performing keywords
3. DataForSEO - Related keywords API
4. Code Node - Filter suggestions:
   - Remove already tracked keywords
   - Filter by minimum search volume
   - Filter by maximum difficulty
5. Code Node - Score and rank suggestions
6. Supabase - Insert top suggestions as "suggested" status
7. Code Node - Format discovery report
8. Slack - Send discoveries to #web-intel
9. Log success
10. Error Handler
