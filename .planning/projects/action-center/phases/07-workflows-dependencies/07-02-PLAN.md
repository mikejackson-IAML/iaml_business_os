# Plan 07-02: Dependency Query Functions

## Goal
Add query functions to fetch the actual task details for blocked-by and blocking relationships, enabling rich dependency visualization.

## Context
The `depends_on` field stores UUID arrays but the UI needs actual task details (title, status) to display meaningful links. The `tasks_extended` view already computes `is_blocked`, `blocked_by_count`, and `blocking_count` but doesn't return the actual task objects.

## Tasks

<task id="1">
Add `getTasksBlockedBy(taskId: string)` function to task-queries.ts:
- Query tasks_extended where id is in the target task's depends_on array
- Return array of TaskExtended objects
- Order by priority, then due_date
</task>

<task id="2">
Add `getTasksBlocking(taskId: string)` function to task-queries.ts:
- Query tasks_extended where depends_on array contains the target task id
- Filter to only incomplete tasks (status NOT IN 'done', 'dismissed')
- Return array of TaskExtended objects
- Order by priority, then due_date
</task>

<task id="3">
Add `getTaskDependencies(taskId: string)` function that returns both:
- blockedBy: TaskExtended[] (tasks this task depends on)
- blocking: TaskExtended[] (tasks waiting on this task)
</task>

<task id="4">
Add `getTaskDependenciesAction(taskId: string)` server action in workflow-actions.ts that wraps getTaskDependencies for client-side use.
</task>

## Verification
- [ ] `getTasksBlockedBy` returns correct task objects for dependencies
- [ ] `getTasksBlocking` returns only incomplete dependent tasks
- [ ] `getTaskDependencies` returns both directions in single call
- [ ] Empty arrays returned when no dependencies exist
- [ ] TypeScript compiles without errors

## Must Haves
- must_have_1: Query returns full TaskExtended objects with all display fields
- must_have_2: Blocking query filters to incomplete tasks only
- must_have_3: Handles tasks with empty depends_on arrays gracefully
