# Plan 07-03: Workflow List Page Foundation

## Goal
Create the workflow list page with route structure, skeleton loading state, and content wrapper following the established SOP list pattern.

## Context
Following the exact pattern from `sops/page.tsx`, `sop-list-skeleton.tsx`, `sop-list-data-loader.tsx`, and `sop-list-content.tsx`. The workflow list uses a table format (matching task list) rather than category groups.

## Tasks

<task id="1">
Create `workflows/page.tsx` with:
- Metadata for title "Workflows | Action Center"
- Suspense wrapper with skeleton fallback
- WorkflowListDataLoader async component
</task>

<task id="2">
Create `workflow-list-skeleton.tsx` with:
- Header skeleton (title + create button)
- Filter bar skeleton (status + department dropdowns)
- Table skeleton with 5 rows matching WorkflowRow dimensions
</task>

<task id="3">
Create `workflow-list-data-loader.tsx` that:
- Calls getWorkflowsAction with default params
- Handles errors with user-friendly message
- Passes workflows to WorkflowListContent
</task>

<task id="4">
Create `workflow-list-content.tsx` shell with:
- Header with title, count, and "New Workflow" button (placeholder link)
- Filter section with status and department dropdowns (state only, no filtering yet)
- Placeholder div for WorkflowTable (to be built in 07-04)
- Back link to Action Center
</task>
