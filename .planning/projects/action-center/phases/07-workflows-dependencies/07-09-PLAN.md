# Plan 07-09: Add Task to Workflow Modal

## Goal
Create a modal to add existing tasks to a workflow, allowing users to build workflows from available tasks.

## Context
WF-05 requires "Add task to workflow action". The API endpoint POST /api/workflows/:id/tasks exists. Need UI to select a task and add it. Show tasks not already in a workflow.

## Tasks

<task id="1">
Create `add-task-to-workflow-modal.tsx` with:
- Props: workflowId, isOpen, onClose, onSuccess
- Modal overlay with Card content (following DismissTaskDialog pattern)
- Header: "Add Task to Workflow"
</task>

<task id="2">
Add task search/selection:
- Search input to filter available tasks
- List of tasks (not in any workflow or in this workflow already)
- Each row: priority dot, title, due date, department
- Click to select, highlight selected
- Maximum 10 results shown
</task>

<task id="3">
Fetch available tasks:
- Call tasks API with filter: workflow_id IS NULL
- Exclude tasks already in this workflow
- Sort by priority, then due_date
- Use client-side fetch with loading state
</task>

<task id="4">
Add task action:
- Submit button enabled when task selected
- Calls addTaskToWorkflowAction(workflowId, selectedTaskId)
- Shows loading state during submission
- Closes modal and calls onSuccess on success
- Shows error message on failure
</task>

<task id="5">
Add "Add Task" button to workflow-detail-content.tsx header:
- Button with Plus icon
- Opens AddTaskToWorkflowModal
- Refreshes page on success (router.refresh)
</task>

<task id="6">
Add export to components/index.ts:
- export { AddTaskToWorkflowModal } from "./add-task-to-workflow-modal";
</task>

## Verification
- [ ] Modal opens from workflow detail page
- [ ] Search filters available tasks
- [ ] Only shows tasks not in a workflow
- [ ] Can select and add task to workflow (WF-05)
- [ ] Workflow task list updates after adding
- [ ] Error handling shows user-friendly message
- [ ] Modal closes on success or cancel

## Must Haves
- must_have_1: Can add existing task to workflow (WF-05)
- must_have_2: Only shows tasks not already in a workflow
- must_have_3: Search functionality to find tasks easily
- must_have_4: Success updates the workflow task list
