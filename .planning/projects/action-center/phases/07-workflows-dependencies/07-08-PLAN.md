# Plan 07-08: Enhanced Dependency Section

## Goal
Replace the minimal dependency count display with a full dependency section showing actual task links for "Blocked by" and "Blocking" with status indicators.

## Context
Per 07-CONTEXT and DEP-04/DEP-05: Show both directions. "Blocked By" lists tasks that must complete first with status badges. "Blocking" lists tasks waiting on this one. Each item is a clickable link.

## Tasks

<task id="1">
Create `task-dependencies.tsx` component with:
- Props: taskId, blockedByCount, blockingCount (to know if we need to fetch)
- Two collapsible sections: "Blocked By" and "Blocking"
- Initially expanded if any dependencies exist
</task>

<task id="2">
Add data fetching using getTaskDependenciesAction:
- Fetch on mount if blockedByCount > 0 or blockingCount > 0
- Loading state while fetching
- Error handling with retry button
</task>

<task id="3">
Build "Blocked By" section:
- Header: "Blocked By" with count badge
- List of tasks this depends on
- Each row: status icon + task title (link) + status badge
- Completed dependencies show checkmark (green)
- Incomplete dependencies show clock/circle (amber)
- Show "All dependencies complete" if blockedBy is empty but was expected
</task>

<task id="4">
Build "Blocking" section:
- Header: "Blocking" with count badge
- List of tasks waiting on this one
- Each row: status icon + task title (link) + status badge
- Helps user understand impact of completing/dismissing this task
</task>

<task id="5">
Update task detail page.tsx to pass blockedByCount and blockingCount to content.
</task>

<task id="6">
Update task-detail-content.tsx:
- Replace minimal dependency display with TaskDependencies component
- Position in main content column (same location as current Dependencies card)
</task>

<task id="7">
Add export to components/index.ts:
- export { TaskDependencies } from "./task-dependencies";
</task>

## Verification
- [ ] Blocked By section shows actual task links (DEP-04)

- [ ] Status icons indicate complete vs incomplete
- [ ] Clicking task link navigates to that task
- [ ] Empty state handled (no dependencies)

- must_have_1: Shows actual task objects, not just counts (DEP-04, DEP-05)

- must_have_3: Each task is a clickable link to its detail page
- must_have_4: Status clearly distinguishes complete vs incomplete dependencies
