# Plan 07-10: Dismiss with Dependents Dialog

## Goal
Enhance dismiss flow to handle tasks that have dependents, with options to dismiss and unblock or create a decision task for handling the cascade.

## Context
Per DEP-06 and 07-CONTEXT: Dismissing task with dependents creates decision task. Modal shows affected tasks and options: "Dismiss and unblock dependents" or "Dismiss and create decision task".

## Tasks

<task id="1">
Create `dismiss-with-dependents-dialog.tsx` with:
- Props: taskId, taskTitle, blockingCount, blockingTasks, isOpen, onClose
- Modal overlay following existing pattern
- Header: "Dismiss Task with Dependents"
</task>

<task id="2">
Build warning section:
- Alert icon with amber styling
- Message: "This task is blocking X other tasks"
- Collapsible list showing affected task titles with links
</task>

<task id="3">
Add option selection:
- Radio buttons for two options:
  1. "Dismiss and unblock dependents" - Dependents will no longer be blocked
  2. "Dismiss and create decision task" - Creates task to handle what to do with dependents
- Default to option 2 (safer)
</task>

<task id="4">
Add standard dismiss fields (reuse from DismissTaskDialog):
- Reason dropdown (required)
- Additional notes textarea (optional)
</task>

<task id="5">
Implement dismiss actions:
- Option 1: Call dismissTaskAction (dependencies naturally unblock when task is done/dismissed)
- Option 2:
  - Call dismissTaskAction
  - Then call createDecisionTaskAction with:
    - title: "Handle dismissed dependency: [task title]"
    - description: "The task '[title]' was dismissed. Review and decide what to do with the following dependent tasks: [list]"
    - priority: same as dismissed task
    - department: same as dismissed task
    - workflow_id: same as dismissed task
    - parent_task_id: dismissed task id
</task>

<task id="6">
Update dismiss-task-dialog.tsx:
- Add blockingCount prop
- If blockingCount > 0, show different UI or redirect to DismissWithDependentsDialog
</task>

<task id="7">
Update task-detail-content.tsx:
- Pass blocking_count to DismissTaskDialog
- Or conditionally render DismissWithDependentsDialog when blocking_count > 0
- Fetch blocking tasks when opening dismiss and blockingCount > 0
</task>

<task id="8">
Add export to components/index.ts:
- export { DismissWithDependentsDialog } from "./dismiss-with-dependents-dialog";
</task>

## Verification
- [ ] Regular dismiss dialog works for tasks without dependents
- [ ] Enhanced dialog shows when task has blocking_count > 0
- [ ] Lists affected tasks in the dialog
- [ ] Option 1 dismisses task normally (dependents unblock automatically)
- [ ] Option 2 dismisses task AND creates decision task (DEP-06)
- [ ] Decision task has correct fields and links to dismissed task
- [ ] Soft enforcement: can proceed with warning (DEP-03)

## Must Haves
- must_have_1: Dismissed task with dependents creates decision task option (DEP-06)
- must_have_2: Shows affected tasks before confirming dismiss
- must_have_3: Soft enforcement - warning but can proceed (DEP-03)
- must_have_4: Decision task contains context about what was dismissed and affected tasks
