# Plan 07-05: Workflow Detail Page Skeleton

## Goal
Create the workflow detail page route structure with loading skeleton and not-found handling.

## Context
Following the pattern from `tasks/[id]/page.tsx` and `sops/[id]/page.tsx`. The detail page will show workflow header with progress, then task list below.

## Tasks

<task id="1">
Create `workflows/[id]/page.tsx` with:
- Dynamic route params for workflow id
- Metadata with workflow name in title
- Suspense with WorkflowDetailSkeleton fallback
- Async component that calls getWorkflowDetailAction
- NotFound if workflow doesn't exist
- Renders WorkflowDetailContent (placeholder div initially)
</task>

<task id="2">
Create `workflow-detail-skeleton.tsx` with:
- Back link skeleton
- Header area: title skeleton, status badge skeleton, progress indicator skeleton
- Metadata row: dates, department
- Task list skeleton: 5 task row skeletons with indentation variations
</task>

<task id="3">
Create `not-found.tsx` with:
- "Workflow Not Found" message
- Link back to workflow list
- Consistent styling with other not-found pages
</task>

<task id="4">
Add data fetching logic in page.tsx:
- Call getWorkflowDetailAction(params.id)
- Handle null response with notFound()
- Pass workflow data to content component
</task>

## Verification
- [ ] Route `/dashboard/action-center/workflows/[id]` renders
- [ ] Skeleton displays during loading
- [ ] 404 page shows for invalid workflow id
- [ ] Valid workflow id shows data (placeholder content for now)
- [ ] Back link navigates to workflow list

## Must Haves
- must_have_1: Dynamic route correctly extracts workflow id from URL
- must_have_2: Loading skeleton matches final layout dimensions
- must_have_3: Not found page has clear navigation back to list
