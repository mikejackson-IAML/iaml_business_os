# Plan 07-04: Workflow Table and Row Components

## Goal
Build WorkflowTable and WorkflowRow components following TaskTable/TaskRow patterns, then integrate into the list content.

## Context
Per 07-CONTEXT decisions: table rows format (not cards), columns: Status, Name, Progress, Due, Department. Progress shown as "3/5 complete" text.

## Tasks

<task id="1">
Create `workflow-table.tsx` with:
- Card container with header row: Status, Name, Progress, Due, Department
- Grid layout: `grid-cols-[80px_1fr_140px_120px_120px]`
- Maps over workflows to render WorkflowRow
- EmptyState component for no results (FileQuestion icon, clear filters action)
</task>

<task id="2">
Create `workflow-row.tsx` with:
- Link wrapper to `/dashboard/action-center/workflows/${workflow.id}`
- Status column: colored circle (green=completed, blue=in_progress, yellow=blocked, gray=not_started)
- Name column: workflow name, description truncated on second line
- Progress column: "X of Y complete" text
- Due column: target_completion_date formatted with overdue highlighting
- Department column: department name or dash
- Hover state: `hover:bg-muted/50`
</task>

<task id="3">
Create status configuration object:
```typescript
const workflowStatusConfig = {
  not_started: { color: 'bg-gray-400', text: 'Not Started' },
  in_progress: { color: 'bg-blue-500', text: 'In Progress' },
  blocked: { color: 'bg-amber-500', text: 'Blocked' },
  completed: { color: 'bg-emerald-500', text: 'Completed' },
};
```
</task>

<task id="4">
Add exports to components/index.ts:
- export { WorkflowTable } from "./workflow-table";
- export { WorkflowRow } from "./workflow-row";
</task>

<task id="5">
Update workflow-list-content.tsx to:
- Import and render WorkflowTable
- Implement filter state management (status[], department string)
- Filter workflows client-side based on filter state
- Pass filtered workflows to WorkflowTable
</task>
