# Plan 07-01: Workflow Server Actions

## Goal
Create server actions for workflow operations: fetching workflows, getting workflow detail, and adding tasks to workflows.

## Context
Phase 3 created the API layer (queries and mutations) but we need server actions for the UI components to call. Following the pattern established in `actions.ts` for tasks.

## Tasks

<task id="1">
Create `workflow-actions.ts` in `/dashboard/src/app/dashboard/action-center/` with the following server actions:
- `getWorkflowsAction(params: WorkflowListParams)` - wraps listWorkflows
- `getWorkflowDetailAction(id: string)` - wraps getWorkflowDetail
- `addTaskToWorkflowAction(workflowId: string, taskId: string)` - wraps addTaskToWorkflow with revalidation
</task>

<task id="2">
Add `createDecisionTaskAction(data: CreateDecisionTaskRequest)` server action that:
- Creates a task with task_type='decision'
- Sets source='workflow'
- Accepts parent_task_id and related_entity fields
- Revalidates action-center paths
</task>

<task id="3">
Define `CreateDecisionTaskRequest` interface with fields:
- title: string
- description: string
- priority: string
- department: string
- workflow_id: string | null
- parent_task_id: string
- related_entity_type: 'task'
- related_entity_id: string
</task>

## Verification
- [ ] `getWorkflowsAction` returns paginated workflow list
- [ ] `getWorkflowDetailAction` returns workflow with tasks
- [ ] `addTaskToWorkflowAction` successfully adds task and revalidates
- [ ] `createDecisionTaskAction` creates decision task with correct fields
- [ ] TypeScript compiles without errors

## Must Haves
- must_have_1: Server actions follow existing pattern from actions.ts (ActionResult return type)
- must_have_2: All actions handle errors gracefully with user-friendly messages
- must_have_3: Path revalidation includes /dashboard/action-center and workflow routes
