# Plan 08-03: Alert-to-Task n8n Workflow Skeleton

## Objective

Create the core n8n workflow that receives alert webhooks and routes them to task creation. This establishes the workflow structure with webhook trigger, severity routing, and database interactions.

## Context

From 08-RESEARCH.md:
- Standard webhook pattern: Webhook → Transform → Dedupe check → Create task → Update alert
- Task API: POST /api/tasks with X-API-Key header
- Dedupe query: Check tasks with same dedupe_key in open/in_progress status

From 08-CONTEXT.md:
- Info alerts: Default do not create tasks (handled by routing)
- Critical/Warning: Create tasks with appropriate priority

## Tasks

<task id="1">
Create n8n workflow structure with these nodes:

**1. Webhook Trigger Node** (`alert-webhook`)
- Path: `/webhook/alert-to-task`
- Method: POST
- Response mode: Last Node

**Expected Webhook Payload:**
```json
{
  "alert_type": "ssl_expiry",
  "severity": "critical",
  "title": "SSL Certificate Expiring Soon",
  "description": "Certificate for www.iaml.com expires in 7 days",
  "affected_resource": "www.iaml.com",
  "source_system": "ssl_monitor",
  "source_alert_id": "uuid-or-id-from-source",
  "metadata": {
    "expiry_date": "2026-02-01",
    "days_remaining": 7
  }
}
```

**2. Validate Payload Node** (`validate-payload`)
- Code node to validate required fields
- Return error if missing alert_type, severity, title, or affected_resource

**3. Get Alert Config Node** (`get-config`)
- Postgres query: `SELECT * FROM action_center.alert_config WHERE alert_type = $1`
- Handle missing config gracefully (use defaults)

**4. Route by Severity Node** (`severity-router`)
- IF node with branches:
  - Critical → critical-path
  - Warning → warning-path
  - Info → info-path

**5. Record Occurrence Node** (`record-occurrence`)
- All paths: Call `action_center.record_alert_occurrence()`
- Store occurrence_id for later

**6. Info Path Handler** (`info-handler`)
- Check if config.info_creates_task = true OR accumulation threshold reached
- If neither, end workflow (no task created)
- If either, continue to dedupe check

**7. Dedupe Check Node** (`check-dedupe`)
- Postgres query to check for existing open/in_progress task
- Query: See dedupe logic in 08-05-PLAN

**8. Priority Escalation Node** (`escalate-priority`)
- If duplicate exists and new severity is higher, update existing task priority
- Continue to "alert processed" path

**9. Create Task Placeholder** (`create-task-placeholder`)
- Placeholder for AI transformation + task creation (08-04)
- For now: Set node with task payload structure

**10. Update Alert Status Node** (`update-alert-status`)
- If source_alert_id provided, update source alert to 'processed'
- Generic: Store in alert_occurrences that task was created

**11. Success Response Node** (`success-response`)
- Return: `{ "success": true, "task_id": "...", "action": "created|escalated|skipped" }`

**12. Error Handler** (`error-handler`)
- Catch all errors
- Log to Slack #workflow-alerts
- Return error response
</task>
