# Plan 05-03: Task Detail Content

## Objective
Create the main task detail content component with two-column layout, header with actions, and tabs for comments/activity.

## Tasks

<task id="1">
Create `task-detail-content.tsx` as a client component with imports:

- useState from 'react'
- Link from 'next/link'
- ArrowLeft from 'lucide-react'
- Button, Card, CardContent from dashboard-kit
- Tabs, TabsList, TabsTrigger, TabsContent from dashboard-kit
- TaskExtended, TaskComment, TaskActivity types
- Component imports (to be created in other plans): TaskMetadataSidebar, TaskComments, TaskActivityList, CompleteTaskDialog, DismissTaskDialog, ApprovalActions, WorkflowContext
</task>

<task id="2">
Implement state management:

- showCompleteDialog: boolean for complete action dialog
- showDismissDialog: boolean for dismiss action dialog
</task>

<task id="3">
Implement header section:

- Back link to `/dashboard/action-center` with ArrowLeft icon
- Task title as h1
- Badges for status, type, priority (reuse config from task-row)
- Action buttons container (right side):
  - For approval tasks: show ApprovalActions component
  - For other tasks: show "Start Working", "Complete", "Dismiss" buttons
  - Buttons trigger appropriate dialogs
</task>

<task id="4">
Implement two-column grid layout (`grid-cols-1 lg:grid-cols-3 gap-6`):

**Left column (lg:col-span-2)**:
- Description card (if task.description exists)
- If task has recommendation (approval task): show RecommendationCallout
- If task has workflow: show WorkflowContext component
- Dependencies section (if task has blocked_by_count > 0 or blocking_count > 0)
- Tabs with Comments and Activity content

**Right column**:
- TaskMetadataSidebar component receiving task
</task>

<task id="5">
Implement the Tabs section for Comments and Activity using Radix Tabs from dashboard-kit:

- defaultValue="comments"
- TabsTrigger for "Comments ({count})" and "Activity"
- TabsContent for TaskComments and TaskActivityList
</task>

<task id="6">
Add conditional rendering:

- Show Complete/Dismiss buttons only for open/in_progress tasks
- For approval tasks, hide Complete/Dismiss and show ApprovalActions instead
- Include CompleteTaskDialog and DismissTaskDialog components at bottom
</task>
