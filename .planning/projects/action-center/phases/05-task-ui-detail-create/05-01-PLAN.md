# Plan 05-01: Server Actions

## Objective
Create server actions file with all task mutation functions for the detail page and create modal.

## Tasks

<task id="1">
Create `dashboard/src/app/dashboard/action-center/actions.ts` with the following server actions:

1. Import pattern from `faculty-scheduler/actions.ts`:
   - 'use server' directive at top
   - Import `revalidatePath` from 'next/cache'
   - Import mutation functions from `@/lib/api/task-mutations`
   - Define `ActionResult` interface: `{ success: boolean; error?: string; data?: Record<string, unknown> }`

2. Implement `completeTaskAction(id: string, note: string | null)`:
   - Call `completeTask` from task-mutations
   - On success: revalidatePath('/dashboard/action-center'), return { success: true }
   - On error: return { success: false, error: error.message }

3. Implement `dismissTaskAction(id: string, reason: string, notes: string | null)`:
   - Validate reason is not empty
   - Combine reason and notes into dismissed_reason string (format: "reason: notes" if notes provided)
   - Call `dismissTask` from task-mutations
   - Revalidate and return result

4. Implement `updateTaskStatusAction(id: string, status: TaskStatus)`:
   - Call `updateTask` with { status }
   - Revalidate and return result

5. Implement `addCommentAction(taskId: string, content: string)`:
   - Validate content is not empty
   - Call `addTaskComment` from task-mutations with authorName 'User' (single-user system)
   - Revalidate path '/dashboard/action-center' and '/dashboard/action-center/tasks/[taskId]'
   - Return { success: true, data: { commentId: comment.id } }

6. Implement `createTaskAction(data: CreateTaskRequest)`:
   - Validate title is not empty
   - Call `createTask` from task-mutations
   - Revalidate and return { success: true, data: { taskId: task.id } }

7. Implement `approveTaskAction(id: string, outcome: 'approved' | 'modified' | 'rejected', modifications: string | null)`:
   - Call updateTask with { approval_outcome: outcome, approval_modifications: modifications, status: 'done' }
   - Note: rejected tasks should also be marked done (decision made)
   - Revalidate and return result
</task>
