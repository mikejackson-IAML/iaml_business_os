# Plan 10-06: Notification Preferences Form in Settings

## Objective

Add notification preferences section to the Settings page allowing users to configure daily digest, digest time, and critical alerts.

## Context

The settings page at `dashboard/src/app/settings/page.tsx` currently shows profile settings (email, full name, role). We need to add a "Notifications" section with:
- Daily Digest toggle (on/off)
- Digest Time picker (time input, only shown when digest is on)
- Critical Alerts toggle (on/off)

The updateProfile function in profiles.ts needs to be extended to handle these new fields.

<tasks>
<task id="1">
Update Profile type in `dashboard/src/lib/supabase/types.ts` to include:
- notification_daily_digest: boolean
- notification_digest_time: string
- notification_critical_alerts: boolean
- timezone: string
</task>
<task id="2">
Update `updateProfile` function in `dashboard/src/lib/supabase/profiles.ts` to accept notification fields
</task>
<task id="3">
Add Notifications section to settings page with:
- Section header "Notifications"
- Daily Digest toggle with label and description
- Digest Time input (type="time") - conditionally shown when digest is on
- Critical Alerts toggle with label and description
</task>
<task id="4">
Create toggle switch component or use a styled checkbox for the on/off toggles
</task>
<task id="5">
Add state management for notification preferences in the settings page
</task>
<task id="6">
Save notification preferences when Save Changes button is clicked
</task>
<task id="7">
Show success/error toast using Sonner when preferences are saved
</task>
</tasks>

## Verification

- [ ] Notifications section appears on Settings page
- [ ] Daily Digest toggle works and persists to database
- [ ] Digest Time input only shows when Daily Digest is enabled
- [ ] Critical Alerts toggle works and persists to database
- [ ] Toast appears on successful save
