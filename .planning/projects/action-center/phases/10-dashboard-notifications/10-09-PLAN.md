# Plan 10-09: n8n Workflow for Daily Digest Scheduling

## Objective

Create an n8n workflow that triggers daily at a configurable time and calls the digest API endpoint to send emails to all eligible users.

## Context

The workflow needs to:
1. Run on a schedule (configurable, default 7am CT)
2. Call the dashboard API endpoint with "all" mode
3. Log results to the n8n dashboard logging system
4. Send Slack alert on failures

The API endpoint handles timezone-aware filtering internally, so the workflow can run once per hour and the endpoint will only send to users whose preferred time matches.

<tasks>
<task id="1">
Create n8n workflow "Daily Digest Sender":
- Schedule trigger: Run every hour (to catch all timezone-based delivery times)
- Alternative: Run at specific times (6am, 7am, 8am, 9am for common preferences)
</task>
<task id="2">
Add HTTP Request node to call digest API:
- Method: POST
- URL: ${DASHBOARD_URL}/api/digest/send
- Headers: X-API-KEY: ${DIGEST_API_KEY}
- Body: { "all": true }
</task>
<task id="3">
Add error handling:
- On HTTP error, send Slack notification
- Log error details
</task>
<task id="4">
Add success logging:
- Log sent/skipped/failed counts
- Store in n8n_brain.workflow_runs if dashboard logging is enabled
</task>
<task id="5">
Create workflow documentation at `business-os/workflows/README-daily-digest.md`:
- CEO Summary
- What it does
- Schedule
- Required credentials/env vars
- Troubleshooting
</task>
<task id="6">
Register workflow in n8n_brain.workflow_registry with:
- category: 'operations'
- department: 'Action Center'
- trigger_type: 'schedule'
- has_error_handling: true
- has_slack_alerts: true
</task>
</tasks>
