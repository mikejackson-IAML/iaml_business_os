# Plan 10-01: Database Migration for Notification Preferences and Task Count RPC

## Objective

Create database migration that adds notification preference columns to profiles table and an RPC function to efficiently count tasks by category (critical, due today, overdue).

## Context

The profiles table exists at `public.profiles` with columns: id, email, full_name, avatar_url, role, is_active, created_at, updated_at. We need to add notification preference columns for:
- Daily digest on/off (default: on)
- Digest send time (default: 07:00)
- Critical alerts on/off (default: on)
- User timezone (for scheduling)

The task count RPC needs to return counts for the dashboard widget and nav badge in a single efficient query.

<tasks>
<task id="1">
Create migration file `supabase/migrations/20260125_notification_prefs_task_counts.sql`
</task>
<task id="2">
Add notification preference columns to profiles table:
- `notification_daily_digest` BOOLEAN DEFAULT TRUE
- `notification_digest_time` TIME DEFAULT '07:00'
- `notification_critical_alerts` BOOLEAN DEFAULT TRUE
- `timezone` TEXT DEFAULT 'America/Chicago'
</task>
<task id="3">
Create RPC function `action_center.get_task_counts()` that returns:
- critical_count: tasks with priority = 'critical' and status NOT IN ('done', 'dismissed')
- due_today_count: tasks due today with status NOT IN ('done', 'dismissed')
- overdue_count: tasks past due_date with status NOT IN ('done', 'dismissed')
- total_active_count: all tasks with status NOT IN ('done', 'dismissed')
- badge_count: critical_count + overdue_count (for nav badge)
</task>
<task id="4">
Create index on tasks table for due_date queries if not exists
</task>
<task id="5">
Add comment documentation for new columns and function
</task>
</tasks>
