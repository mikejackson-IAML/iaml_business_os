# Plan 10-04: Navigation Badge with Real-time Subscription

## Objective

Create a badge component for the Action Center navigation item that shows the count of critical + overdue tasks, updating in real-time via Supabase subscription.

## Context

The badge shows (critical_count + overdue_count) from the task counts RPC. It should:
- Update in real-time when tasks change
- Hide when count is 0
- Display "9+" when count exceeds 9

The badge will be added to the Action Center quick link on the main dashboard header area (in dashboard-content.tsx).

Real-time updates use Supabase's realtime subscriptions on the tasks table.

<tasks>
<task id="1">
Create custom hook `useTaskBadgeCount()` at `dashboard/src/hooks/use-task-badge-count.ts`:
- Fetches initial count via getTaskCounts()
- Subscribes to action_center.tasks table changes
- Refetches count on INSERT, UPDATE, DELETE events
- Returns { count, isLoading }
- Cleans up subscription on unmount
</task>
<task id="2">
Create badge component at `dashboard/src/components/nav/action-center-badge.tsx`:
- Renders count in a small red circle
- Shows "9+" when count > 9
- Hidden when count is 0 or loading
- Uses absolute positioning for overlay on parent
</task>
<task id="3">
Style badge with: bg-red-500, text-white, text-xs, rounded-full, min-w-[1.25rem], h-5
</task>
<task id="4">
Add badge to Action Center link in dashboard-content.tsx header quick links section
</task>
<task id="5">
Handle subscription errors gracefully with console.error and fallback to polling if subscription fails
</task>
</tasks>

## Verification

- [ ] Badge displays correct count (critical + overdue)
- [ ] Badge hidden when count is 0
- [ ] Badge shows "9+" when count exceeds 9
- [ ] Creating a critical task updates badge without page refresh
- [ ] Completing a critical task updates badge without page refresh
- [ ] Subscription cleans up properly on component unmount
- [ ] No memory leaks or duplicate subscriptions
