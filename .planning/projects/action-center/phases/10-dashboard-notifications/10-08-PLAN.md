# Plan 10-08: Digest Generation Function and API Endpoint

## Objective

Create an API endpoint that generates and sends the daily digest email, designed to be called by an n8n workflow scheduler.

## Context

The API endpoint will be called by n8n at the appropriate time for each user based on their timezone and preferred digest time. The endpoint needs to:
1. Accept a user ID (or "all" to process all users due for digest)
2. Check notification preferences
3. Generate digest data
4. Send email via Resend
5. Return success/failure status

For security, the endpoint should require an API key header.

<tasks>
<task id="1">
Create digest data generator at `dashboard/src/lib/email/generate-digest-data.ts`:
- Function `generateDigestData(userId: string)` that returns:
  - criticalTasks: array of {id, title, due_date}
  - dueTodayTasks: array of {id, title, due_date}
  - overdueTasks: array of {id, title, due_date}
  - stats: {totalActive, completedThisWeek}
- Uses server-side Supabase client
</task>
<task id="2">
Create API route at `dashboard/src/app/api/digest/send/route.ts`:
- POST handler
- Accepts JSON body: { userId: string } or { all: true }
- Validates API key from X-API-KEY header against DIGEST_API_KEY env var
</task>
<task id="3">
Implement single-user digest logic:
- Fetch user profile and notification prefs
- Skip if notification_daily_digest is false
- Generate digest data
- Send email
- Return result
</task>
<task id="4">
Implement batch processing for "all" mode:
- Query users where notification_daily_digest = true
- Filter by current time matching their digest_time + timezone
- Process each user
- Return summary of sent/skipped/failed
</task>
<task id="5">
Add rate limiting consideration: process in batches of 10 with small delays
</task>
<task id="6">
Return structured JSON response:
- { success: true, sent: number, skipped: number, failed: number }
- { success: false, error: string }
</task>
</tasks>

## Verification

- [ ] API endpoint responds to POST requests
- [ ] Rejects requests without valid API key
- [ ] Single user mode sends digest to specific user
- [ ] Batch mode processes all eligible users
- [ ] Skips users with digest disabled
- [ ] Returns accurate counts in response
- [ ] Handles errors gracefully without crashing

## must_haves

- API endpoint at `dashboard/src/app/api/digest/send/route.ts`
- Digest data generator at `dashboard/src/lib/email/generate-digest-data.ts`
- API key authentication
- Batch processing support
