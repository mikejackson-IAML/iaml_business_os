## Goal

Create TypeScript query functions to fetch metrics from existing database views.

## Requirements Covered

- MET-01 (personal stats)
- MET-02 (system overview)
- MET-03 (department metrics)

## Context

The Action Center already has these views ready to use:
- `action_center.user_task_summary` - Personal stats per user
- `action_center.department_task_summary` - Department aggregations
- `action_center.system_task_summary` - System-wide metrics

This plan creates the TypeScript layer to query them.

## Tasks

<task id="1">
Create `dashboard/src/lib/api/task-metrics.ts` with type definitions:

```typescript
export interface PersonalStats {
  completed_today: number;
  completed_this_week: number;
  weekly_goal_target: number | null;
  weekly_goal_progress: number | null; // percentage 0-100
  overdue_count: number;
  actionable_count: number;
  open_count: number;
  in_progress_count: number;
  avg_completion_days_30d: number | null;
}

export interface DepartmentMetrics {
  department: string;
  open_count: number;
  overdue_count: number;
  actionable_count: number;
  critical_count: number;
  high_priority_count: number;
  created_last_7_days: number;
  completed_last_7_days: number;
  completion_rate_30d: number | null;
  avg_completion_days_30d: number | null;
}

export interface SystemMetrics {
  total_open: number;
  total_overdue: number;
  total_actionable: number;
  critical_count: number;
  high_priority_count: number;
  created_last_7_days: number;
  completed_last_7_days: number;
  completion_rate_7d: number | null;
}
```
</task>

<task id="2">
Add `getPersonalStats(userId: string): Promise<PersonalStats>` function that:
- Queries `user_task_summary` view filtered by user_id
- Calls `get_completed_today_count` RPC for today's count
- Calls `get_weekly_goal_progress` RPC for goal data
- Merges into PersonalStats shape
- Handle null user gracefully (return zeros)
</task>

<task id="3">
Add `getDepartmentMetrics(department: string): Promise<DepartmentMetrics>` function that:
- Queries `department_task_summary` view filtered by department
- Returns DepartmentMetrics shape
- Throws if department not found
</task>

<task id="4">
Add `getAllDepartmentMetrics(): Promise<DepartmentMetrics[]>` function that:
- Queries `department_task_summary` view (all rows)
- Returns array sorted by department name
</task>

<task id="5">
Add `getSystemMetrics(): Promise<SystemMetrics>` function that:
- Queries `system_task_summary` view
- Returns SystemMetrics shape
- This is a single-row view
</task>

<task id="6">
Export all functions and types from the module. Add JSDoc comments explaining each function's purpose.
</task>
