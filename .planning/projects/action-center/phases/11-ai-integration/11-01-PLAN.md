# Plan 11-01: AI Analysis API Endpoint

## Goal
Create the backend API endpoint that fetches 90-day task history, calls Claude API for analysis, and returns structured suggestions.

## Requirements Covered
- AI-02: AI analyzes open tasks, overdue items, patterns (90-day lookback)
- AI-08: Pattern detection (partial - data preparation)

## Context
- Claude API integration exists in `/dashboard/src/lib/api/mobile-chat.ts`
- Tasks table has ai_confidence, ai_suggested_at columns (no schema changes needed)
- Use Claude Opus for best quality (weekly analysis, not realtime)

<title>Create AI analysis utility functions</title>
<file path="dashboard/src/lib/action-center/ai-analysis.ts" action="create" />
<description>
Create analysis utilities:
1. fetchAnalysisData(userId: string, lookbackDays: number = 90): Promise<TaskExtended[]>
   - Query tasks from last 90 days for user
   - Include status, priority, due dates, completed_at, dismissed_at, dismissed_reason

2. buildAnalysisPrompt(tasks: TaskExtended[], isWeeklyPlanning: boolean): string
   - Format task data as structured JSON for Claude
   - Include system prompt with "encouraging coach" tone
   - Distinguish between Sunday planning vs Friday recap mode

3. parseAnalysisResponse(response: string): AIAnalysisResult
   - Parse Claude's JSON response
   - Validate structure and handle malformed responses
   - Extract suggestions with confidence scores

4. callClaudeAPI(prompt: string, systemPrompt: string): Promise<string>
   - Use Anthropic SDK (claude-opus-4-5-20251101)
   - max_tokens: 4096
   - Handle rate limits and errors gracefully
</description>
</task>
