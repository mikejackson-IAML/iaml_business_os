# Phase 5 Research: Alerts System

**Researched:** 2026-01-24
**Domain:** Dashboard UI, Server Actions, Web Intelligence
**Confidence:** HIGH

## Summary

This phase adds an alerts management feature to the Web Intelligence dashboard. The foundation is already in place: the `web_intel.alerts` table exists with the required columns (alert_type, severity, title, message, acknowledged_at), TypeScript types are defined (`WebIntelAlert` and `WebIntelAlertDb`), and query functions exist (`getWebIntelAlerts`).

The current implementation shows alerts in a simple card on the Overview tab. This phase will build a dedicated Alerts section with filtering, acknowledgment actions, and proper UI for card-based alert display with severity indicators via icons (per user decision - no border stripes).

**Primary recommendation:** Build a new `AlertsSection` component with filter chips, use server actions for acknowledge/dismiss functionality, and follow the existing `PriorityFilter` pattern for URL-based filter state.

## Schema Analysis

### web_intel.alerts Table Structure

| Column | Type | Purpose |
|--------|------|---------|
| `id` | UUID | Primary key |
| `alert_type` | TEXT | Category: traffic_anomaly, ranking_change, index_error, etc. |
| `severity` | TEXT | `info`, `warning`, `critical` (constraint enforced) |
| `title` | TEXT | Alert headline |
| `message` | TEXT | Detailed description |
| `metadata` | JSONB | Additional context data |
| `source_workflow` | TEXT | Which n8n workflow generated the alert |
| `acknowledged_at` | TIMESTAMPTZ | When dismissed (NULL = active) |
| `acknowledged_by` | TEXT | Who dismissed it |
| `resolved_at` | TIMESTAMPTZ | When underlying issue was resolved |
| `created_at` | TIMESTAMPTZ | When alert was created |

**Indexes:**
- `idx_alerts_type` - For filtering by type
- `idx_alerts_severity` - For filtering by severity
- `idx_alerts_unacknowledged` - Partial index on `acknowledged_at IS NULL` for active alerts
- `idx_alerts_date` - For sorting by creation date

### Alert Types (from schema comments)

The `alert_type` values appear to be:
- `traffic_anomaly` - Unusual traffic patterns
- `ranking_change` - Keyword position changes
- `index_error` - GSC indexing issues
- (Others as generated by workflows)

For filtering, the requirements specify: `traffic`, `ranking`, `technical`. This maps to:
- `traffic` = `traffic_anomaly` type alerts
- `ranking` = `ranking_change` type alerts
- `technical` = `index_error` and similar technical alerts

export async function acknowledgeAlertAction(id: string): Promise<ActionResult> {
  try {
    // Call mutation function

### Mutation Query (new file)

**File:** `dashboard/src/lib/api/web-intel-mutations.ts`

```typescript
export async function acknowledgeAlert(id: string, acknowledgedBy?: string): Promise<void> {
  const supabase = getServerClient();

  const { error } = await supabase
    .from('web_intel.alerts')
    .update({
      acknowledged_at: new Date().toISOString(),
      acknowledged_by: acknowledgedBy ?? 'user',
    })
    .eq('id', id);

  if (error) throw error;
}

export async function acknowledgeAlerts(ids: string[], acknowledgedBy?: string): Promise<number> {
  const supabase = getServerClient();

  const { error, count } = await supabase
    .from('web_intel.alerts')
    .update({
      acknowledged_at: new Date().toISOString(),
      acknowledged_by: acknowledgedBy ?? 'user',
    })
    .in('id', ids);

  if (error) throw error;
  return count ?? 0;
}
```

## Implementation Notes

### CONTEXT.md Decisions to Follow

1. **Card layout** - Vertically stacked cards, no grid
2. **Severity indication** - Icons only, NO left borders or top borders
3. **Message truncation** - 2-line max with ellipsis
4. **Dismiss button** - Hover-only visibility, single click (no confirmation)
5. **Fade animation** - On dismiss, fade out then slide remaining cards up
6. **Filter chips** - Horizontal bar: All | Traffic | Ranking | Technical
7. **Chip counts** - Show count in parentheses: "Traffic (3)"
8. **Empty state** - Simple "No alerts" text, no icon
9. **Loading state** - 2-3 skeleton placeholders with shimmer
10. **Dismiss All** - Single button for all visible alerts (not filtered-only)
