# Plan 04-01: Device Toggle and CWV Metric Components

## Objective

Create the foundational UI components for Core Web Vitals display: a mobile/desktop toggle and an individual CWV metric component with status badge.

## Requirements Covered

- CWV-04: User can toggle between mobile and desktop vitals (device toggle)
- CWV-01, CWV-02, CWV-03: Metrics display with status badges (CWV metric component)

## Tasks

<task id="1">
Create `device-toggle.tsx` in `dashboard/src/app/dashboard/web-intel/components/`.

Component specifications:
- Props: `deviceType: 'mobile' | 'desktop'`, `onDeviceChange: (device: 'mobile' | 'desktop') => void`
- Visual style: Copy from `date-range-selector.tsx` (line 30-45) - same segmented control pattern
- Two buttons: "Mobile" and "Desktop"
- Active button has `bg-background text-foreground shadow-sm`
- Inactive button has `text-muted-foreground hover:text-foreground hover:bg-background/50`
- Use `cn` from `@/dashboard-kit/lib/utils` for class merging
- Mark as `'use client'` since it handles clicks
- Do NOT persist to URL (use local state, per CONTEXT.md decision)
</task>

<task id="2">
Create `cwv-metric.tsx` in `dashboard/src/app/dashboard/web-intel/components/`.

Component specifications:
- Props: `label: string` (e.g., "LCP"), `value: number | null` (the good percentage), `className?: string`
- Derives status from value:
  - `value >= 75`: "Good" (healthy badge)
  - `value >= 50`: "Needs Work" (warning badge)
  - Otherwise: "Poor" (critical badge)
  - If value is null: show "-" with no badge
- Layout: Vertical stack with label on top, value + badge below
- Value format: `{value.toFixed(1)}%` (shows one decimal)
- Import `Badge` from `@/dashboard-kit/components/ui/badge`
- Badge variant mapping: good -> "healthy", needs_work -> "warning", poor -> "critical"
- Label uses `text-sm text-muted-foreground`
- Value uses `text-2xl font-semibold`
</task>
