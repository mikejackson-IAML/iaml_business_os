# 07-01: Health API Endpoint

## Objective

Create the `/api/mobile/health` Next.js API route that returns aggregated department health scores with API key authentication. This is the backend foundation that the iOS app will consume.

**Purpose:** Provide a secure, mobile-optimized endpoint that aggregates health data from Supabase (workflow stats) and external services (uptime, performance) into a unified JSON response.

**Output:** Working API endpoint at `/api/mobile/health` that returns department health data when authenticated with a valid API key.

## Context

The Next.js dashboard at `/dashboard` already has comprehensive data fetching infrastructure:
- `workflow-queries.ts` - Fetches workflow health from n8n_brain schema
- `digital-queries.ts` - Aggregates uptime, performance, database metrics

We need to create a new API route that:
1. Validates API key from `X-API-Key` header
2. Aggregates data from existing query functions
3. Returns a mobile-optimized JSON format

The iOS app (Phase 6 complete) stores API key in Keychain and will pass it via `X-API-Key` header.

**Types to define:**

```typescript
export type HealthStatus = 'healthy' | 'warning' | 'critical';
export type AlertSeverity = 'info' | 'warning' | 'critical';

export interface TopMetric {
  label: string;
  value: string;
  status: HealthStatus;
}

export interface DepartmentHealth {
  id: string;
  name: string;
  score: number;
  status: HealthStatus;
  alertCount: number;
  topMetrics: TopMetric[];
}

export interface HealthAlert {
  id: string;
  title: string;
  severity: AlertSeverity;
  department: string;
  timestamp: string;
}

export interface MobileHealthResponse {
  timestamp: string;
  overallHealth: {
    score: number;
    status: HealthStatus;
  };
  departments: DepartmentHealth[];
  alerts: HealthAlert[];
  totalAlertCount: number;
}
```

**Aggregation function:**

Create `getMobileHealthData()` that:
1. Calls `getWorkflowStats()` from workflow-queries.ts
2. Calls `getDigitalMetrics()` from digital-queries.ts
3. Transforms data into two departments:
   - **Workflows**: score based on successRate, metrics: success rate, unresolved errors
   - **Digital**: score based on uptime and LCP, metrics: uptime %, LCP
4. Generates alerts from data (unresolved errors = warning alerts, critical issues = critical alerts)
5. Calculates overall health as weighted average of department scores

**Score calculation:**
- Workflow score: successRate (0-100 directly maps)
- Digital score: uptime (50%) + LCP score (50%) where LCP score is 100 if <2.5s, 75 if <4s, 50 if <5s, 25 otherwise

**Status thresholds:**
- healthy: score >= 85
- warning: score >= 60
- critical: score < 60

  <name>Task 2: Create API route with authentication</name>
  <files>dashboard/src/app/api/mobile/health/route.ts</files>

Create the Next.js API route handler at `/api/mobile/health`.

**Authentication:**
- Read `X-API-Key` header from request
- Compare against `process.env.MOBILE_API_KEY`
- Return 401 JSON response if missing or invalid: `{ error: "Unauthorized" }`

**Route handler:**

```typescript
import { NextRequest, NextResponse } from 'next/server';
import { getMobileHealthData } from '@/lib/api/mobile-health';

export async function GET(request: NextRequest) {
  // Validate API key
  const apiKey = request.headers.get('X-API-Key');
  const validApiKey = process.env.MOBILE_API_KEY;

  if (!apiKey || !validApiKey || apiKey !== validApiKey) {
    return NextResponse.json(
      { error: 'Unauthorized' },
      { status: 401 }
    );
  }

  try {
    const healthData = await getMobileHealthData();
    return NextResponse.json(healthData);
  } catch (error) {
    console.error('Health API error:', error);
    return NextResponse.json(
      { error: 'Internal server error' },
      { status: 500 }
    );
  }
}
```

**CORS:** No CORS headers needed - iOS app makes same-origin requests to the API.

**Caching:** Add cache headers for 60 second stale-while-revalidate:
```typescript
return NextResponse.json(healthData, {
  headers: {
    'Cache-Control': 'public, s-maxage=60, stale-while-revalidate=120',
  },
});
```
