  <name>Task 1: Create workflow trigger library</name>
  <files>dashboard/src/lib/api/workflow-triggers.ts</files>
  <action>
Create workflow trigger library following mobile-health.ts pattern:

1. `triggerWorkflow(workflowId: string, webhookUrl: string, parameters?: Record<string, unknown>)`:
   - POST to webhookUrl with JSON body (parameters or empty object)
   - Return { success: boolean, executionId?: string, message: string }
   - Handle errors gracefully (webhook may timeout, return non-JSON, etc.)
   - Set 10-second timeout to prevent hanging

2. `getAvailableWorkflows()`:
   - Query n8n_brain.workflow_registry from Supabase
   - Filter: quick_action_enabled = true AND webhook_url IS NOT NULL
   - Select: workflow_id, workflow_name, description, quick_action_icon, risk_level, category
   - Return array mapped to QuickAction format (id, name, description, icon, riskLevel, category, canTrigger)

3. `getWorkflowById(workflowId: string)`:
   - Query single workflow by workflow_id
   - Return null if not found
   - Used by trigger endpoint to get webhook_url

Use existing Supabase client pattern from mobile-health.ts.
  </action>

Create two API route handlers following mobile/health/route.ts pattern:

**GET /api/mobile/workflows (route.ts):**
1. Validate X-API-Key header (same pattern as health endpoint)
2. Call getAvailableWorkflows()
3. Return JSON: { workflows: [...] }
4. Handle errors with appropriate status codes

**POST /api/mobile/workflows/trigger (trigger/route.ts):**
1. Validate X-API-Key header
2. Parse JSON body: { workflow_id: string, parameters?: object }
3. Look up workflow with getWorkflowById(workflow_id)
4. If not found or no webhook_url: return 404 with { error: 'Workflow not found or cannot be triggered' }
5. Call triggerWorkflow(workflow_id, webhook_url, parameters)
6. Return JSON: { success, executionId, message }

Both endpoints:
- Return 401 for missing/invalid API key
- Return 500 for server errors with generic message
- Log errors to console for debugging
