   - If result has error, return 200 with { error, suggestion } (not 4xx -- it's a valid request, AI just couldn't parse)

2. Create `generate-summary/route.ts`:
   - POST handler: extract { contactId, force } from request body (force = regenerate regardless of cache)
   - Validate contactId is UUID
   - Fetch contact from li_contacts (include ai_summary, ai_summary_generated_at)
   - If !force and ai_summary exists and ai_summary_generated_at < 30 days old, return cached: { summary: ai_summary, generated_at: ai_summary_generated_at, cached: true }
   - Otherwise: fetch contact + attendance_records + activities + follow_ups from Supabase
   - Build contactData object with all context
   - Call generateContactSummary(contactData)
   - Save result: UPDATE li_contacts SET ai_summary = result, ai_summary_generated_at = NOW() WHERE id = contactId
   - Return { summary, generated_at: now, cached: false }
   - Use supabase-any-cast pattern for table queries (accumulated decision)
