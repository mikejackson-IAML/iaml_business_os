Create the 4-file API foundation following the exact contacts pattern:

**types.ts:**
- `Opportunity` interface matching the opportunities table schema (id, title, type, stage, company_id, contact_id, value, notes, created_at, updated_at) with optional joined `company?: { id: string; name: string }` and `contact?: { id: string; first_name: string; last_name: string }`
- `OpportunityContact` interface (opportunity_id, contact_id, role, created_at) with optional joined contact data
- `OpportunityAttachment` interface (id, opportunity_id, file_name, file_url, file_type, file_size, uploaded_at)
- `OpportunityListParams` (page, limit, sort, order, type, stage, company_id, search)
- Stage constants: `IN_HOUSE_STAGES` (inquiry, strategy_session, consultation, proposal_sent, planning, won, lost) and `INDIVIDUAL_STAGES` (inquiry, info_sent, follow_up, registered, lost) as const arrays
- Stage label maps: `IN_HOUSE_STAGE_LABELS` and `INDIVIDUAL_STAGE_LABELS` as Record<string, string>
- `CONTACT_ROLES` const array (decision_maker, influencer, champion, end_user, billing_contact)
- `CONTACT_ROLE_LABELS` map

**validation.ts:**
- `validateOpportunity(data)` -- requires title, type must be 'in_house' or 'individual', validates stage against correct pipeline based on type, value must be non-negative if provided
- `validateStageAdvancement(type, newStage)` -- validates newStage exists in correct pipeline array for given type

**queries.ts:**
- `getOpportunitiesTable()` using `getServerClient().from('opportunities') as any` pattern
- `getOpportunities(params)` -- paginated with filters (type, stage, company_id, search via ilike on title), joins companies(id, name) and contacts(id, first_name, last_name), returns {data, meta}
- `getOpportunityById(id)` -- single opportunity with company + contact joins
- `getOpportunityContacts(opportunityId)` -- from opportunity_contacts with contact join (first_name, last_name, email, title)
- `getOpportunityAttachments(opportunityId)` -- from opportunity_attachments ordered by uploaded_at desc

**mutations.ts:**
- `createOpportunity(data)` -- insert with default stage = 'inquiry'
- `updateOpportunity(id, data)` -- partial update with updated_at = now
- `deleteOpportunity(id)` -- delete (cascades to contacts and attachments)
- `advanceStage(id, type, newStage)` -- update stage field
- `addOpportunityContact(opportunityId, contactId, role)` -- insert into opportunity_contacts
- `removeOpportunityContact(opportunityId, contactId)` -- delete from opportunity_contacts
- `createAttachment(data: { opportunity_id, file_name, file_url, file_type, file_size })` -- insert into opportunity_attachments
- `deleteAttachment(id)` -- delete attachment record

Use `as any` cast for all .from() calls (tables not in Database type). Use `as never` for insert/update payloads if needed (same pattern as contacts).

Create 5 route files following the contacts route pattern (validateApiKey from task-auth, try/catch, NextResponse.json):

**opportunities/route.ts:**
- GET: parse query params (page, limit, sort, order, type, stage, company_id, search), call getOpportunities, return paginated response
- POST: validate body with validateOpportunity, call createOpportunity, return 201

**opportunities/[id]/route.ts:**
- GET: call getOpportunityById, return 404 if not found, include contacts and attachments in response
- PATCH: validate partial data, call updateOpportunity
- DELETE: call deleteOpportunity, return 204

**opportunities/[id]/advance-stage/route.ts:**
- POST: body requires { stage }, fetch opportunity to get type, call validateStageAdvancement, call advanceStage, return updated opportunity

**opportunities/[id]/contacts/route.ts:**
- GET: call getOpportunityContacts
- POST: body requires { contact_id, role }, validate role is in CONTACT_ROLES, call addOpportunityContact, return 201
- DELETE: body requires { contact_id }, call removeOpportunityContact, return 204

**opportunities/[id]/attachments/route.ts:**
- GET: call getOpportunityAttachments
- POST: accept FormData with 'file' field. Upload to Supabase Storage bucket 'opportunity-attachments' at path `opportunities/{id}/{timestamp}-{filename}`. Get public URL. Call createAttachment with metadata. Return 201.
  - Set `export const runtime = 'nodejs'` (not edge, for Buffer support)
  - Handle file size > 10MB with 413 error
- DELETE: body requires { attachment_id }, call deleteAttachment (also delete from storage), return 204

For the storage bucket: the upload route should handle the case where the bucket doesn't exist gracefully (create it programmatically using `supabase.storage.createBucket('opportunity-attachments', { public: false })` on first upload failure, or attempt creation at start).

Use signed URLs (not public URLs) since these are proposals/contracts. Generate signed URL valid for 1 hour when returning attachment data.
