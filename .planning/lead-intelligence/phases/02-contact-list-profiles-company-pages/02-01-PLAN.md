  <action>
    Create query modules and API routes for each sub-resource. Follow the exact same pattern as existing contacts/companies routes (validateApiKey, try/catch, NextResponse.json). Use `as any` cast for .from() on new tables per Phase 1 decision.

    **Query modules (in dashboard/src/lib/api/):**

    1. attendance-queries.ts: `getContactAttendance(contactId)` — query `attendance_records` where contact_id matches, order by event_date desc. Select all fields.

    2. email-queries.ts: `getContactEmailActivities(contactId)` — query `email_activities` where contact_id matches, order by activity_date desc. Select all fields.

    3. notes-queries.ts: `getContactNotes(contactId)` and `createContactNote(contactId, data)` — query/insert `contact_notes`. Also `getCompanyNotes(companyId)` and `createCompanyNote(companyId, data)`. Order by created_at desc.

    4. followups-queries.ts: `getContactFollowUps(contactId)` — query `follow_up_tasks` where contact_id matches, order by due_date asc.

    5. activity-queries.ts: `getContactActivity(contactId)` — query `activity_log` where entity_type='contact' and entity_id matches, order by created_at desc, limit 10 (for overview tab recent activity).

    **API routes (all use validateApiKey, await params for [id]):**

    6. contacts/[id]/attendance/route.ts: GET -> getContactAttendance(id)
    7. contacts/[id]/email-activities/route.ts: GET -> getContactEmailActivities(id)
    8. contacts/[id]/notes/route.ts: GET -> getContactNotes(id), POST -> createContactNote(id, body). POST body: { note_type: string, content: string }. Validate note_type is one of: general, call, meeting, email, system.
    9. contacts/[id]/follow-ups/route.ts: GET -> getContactFollowUps(id)
    10. contacts/[id]/activity/route.ts: GET -> getContactActivity(id)
    11. companies/[id]/contacts/route.ts: GET -> query contacts table where company_id matches, paginated.
    12. companies/[id]/notes/route.ts: GET -> getCompanyNotes(id), POST -> createCompanyNote(id, body). Same validation as contact notes.

    All Next.js 16 dynamic route handlers must use `params: Promise<{ id: string }>` and `const { id } = await params;`.
