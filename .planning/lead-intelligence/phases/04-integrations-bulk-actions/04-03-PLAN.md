    Create enrichment-merge.ts:
    - Export `mergeEnrichment(existing, enrichedData)` following the pattern from 04-RESEARCH.md
    - Fields to merge for contacts: title, department, phone, linkedin_url, seniority_level, company_name
    - Fields to merge for companies: industry, employee_count, revenue_range, website, description
    - Returns `{ updates: Partial<Contact>, conflicts: Array<{field, existing, enriched}> }`
    - Fill blanks only: skip if existing value is non-null/non-empty
    - Flag conflicts: when enriched differs from existing, add to conflicts array

    Create contacts/[id]/enrich/route.ts (POST):
    - validateApiKey
    - Fetch contact from Supabase
    - Call n8n webhook `${process.env.N8N_WEBHOOK_BASE_URL || 'https://n8n.realtyamp.ai'}/webhook/enrich-contact` with { email, linkedin_url, name }
    - If n8n returns data: run mergeEnrichment, apply updates to contact, store raw JSON in enrichment_data column, store conflicts in enrichment_data.conflicts
    - Update enrichment tracking: last_enriched_at = now(), enrichment_source = response source
    - Return { success, updates_applied: N, conflicts: N, enriched_fields: [...] }
    - If n8n webhook fails or returns empty: return { success: false, error: 'No enrichment data found' } with 200 (not error — it's expected)

    Create companies/[id]/enrich/route.ts (POST):
    - Same pattern but for company fields
    - Webhook: /webhook/enrich-company with { company_name, domain, website }

    Create bulk/enrich/route.ts (POST):
    - Accept { contactIds: string[] }
    - Process sequentially (not parallel) with 1s delay between calls to avoid rate limits
    - Return { success: true, results: Array<{ contactId, updates_applied, conflicts, error? }> }
