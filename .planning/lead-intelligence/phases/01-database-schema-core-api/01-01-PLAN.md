Create a single SQL migration file with ALL Lead Intelligence tables in this exact order (respecting FK dependencies):

1. **companies** — name, website, industry, employee_count, revenue_range, city/state/country, linkedin_url, enrichment_source, enrichment_data (jsonb), enriched_at, created_at, updated_at. UUID PK with gen_random_uuid().

2. **contacts** — first_name, last_name, email (UNIQUE), phone, linkedin_url, title, department, seniority_level, company_id (FK companies), city/state/country, status (enum or text: lead/customer/prospect/inactive/do_not_contact), classification (enum or text), lead_source, engagement_score (integer default 0), is_vip (boolean default false), email_status (text: valid/invalid/unknown/risky), email_verified_at, last_activity_at, enrichment_source, enrichment_data (jsonb), enriched_at, profile_image_url, linkedin_member_id, linkedin_headline, created_at, updated_at. UUID PK.

3. **attendance_records** — contact_id (FK contacts), program_name (text, NOT FK to programs — avoid FK dependency on potentially missing table), program_date, rating (integer 1-5 nullable), feedback (text nullable), attendance_status (text: attended/no_show/cancelled), created_at. UUID PK.

4. **email_activities** — contact_id (FK contacts), campaign_id (text), campaign_name (text), activity_type (text: sent/open/click/reply/bounce/unsubscribe), activity_data (jsonb), occurred_at, created_at. UUID PK.

5. **opportunities** — title, type (text: in_house/individual), stage (text), company_id (FK companies nullable), contact_id (FK contacts nullable), value (numeric nullable), expected_close_date (date nullable), notes (text nullable), created_at, updated_at. UUID PK.

6. **opportunity_contacts** — opportunity_id (FK opportunities ON DELETE CASCADE), contact_id (FK contacts), role (text: decision_maker/influencer/champion/end_user/blocker), created_at. Composite PK (opportunity_id, contact_id).

7. **opportunity_attachments** — opportunity_id (FK opportunities ON DELETE CASCADE), file_name, file_url, file_type (text), file_size (integer), uploaded_at (default NOW()). UUID PK.

8. **contact_notes** — contact_id (FK contacts ON DELETE CASCADE), note_type (text: general/call/meeting/email/system), content (text), created_at, updated_at. UUID PK.

9. **company_notes** — company_id (FK companies ON DELETE CASCADE), note_type (text: general/call/meeting/email/system), content (text), created_at, updated_at. UUID PK.

10. **activity_log** — entity_type (text: contact/company/opportunity), entity_id (uuid), action (text), details (jsonb), created_at. UUID PK. No foreign keys (polymorphic).

11. **follow_up_tasks** — contact_id (FK contacts nullable), company_id (FK companies nullable), opportunity_id (FK opportunities nullable), title, description (text nullable), due_date (date), status (text: pending/completed/cancelled default pending), priority (text: low/medium/high default medium), action_center_task_id (uuid nullable), completed_at (timestamp nullable), created_at, updated_at. UUID PK.

12. **data_health_metrics VIEW** — aggregates from contacts table:
    - total_contacts (count)
    - valid_emails (count where email_status = 'valid')
    - invalid_emails (count where email_status = 'invalid')
    - unknown_emails (count where email_status = 'unknown' or null)
    - email_health_pct (valid/total * 100)
    - enriched_contacts (count where enriched_at is not null)
    - enrichment_pct (enriched/total * 100)
    - stale_contacts (count where last_activity_at < NOW() - interval '90 days' or null)
    - freshness_pct ((total - stale) / total * 100)
    - completeness_pct (avg of: has_email, has_phone, has_title, has_company per contact * 100)
    - overall_quality_score (avg of email_health_pct, enrichment_pct, freshness_pct, completeness_pct)

13. **set_updated_at() trigger function** (CREATE OR REPLACE) and triggers on: contacts, companies, opportunities, contact_notes, company_notes, follow_up_tasks.

14. **Indexes:**
    - contacts: email, company_id, status, engagement_score, last_activity_at, linkedin_member_id
    - companies: name, industry
    - attendance_records: contact_id, program_name
    - email_activities: contact_id, campaign_id, activity_type
    - opportunities: company_id, contact_id, stage, type
    - activity_log: entity_type + entity_id (composite), created_at
    - follow_up_tasks: contact_id, company_id, due_date, status

Use text types (not enums) for status fields — easier to extend without migrations.
Avoid FK to programs table (use program_name text instead per research pitfall #1).
