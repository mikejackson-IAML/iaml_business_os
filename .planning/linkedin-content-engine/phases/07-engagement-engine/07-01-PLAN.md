Build the WF6 Engagement Engine n8n workflow and its required schema migration. Purpose: Enable strategic LinkedIn engagement by surfacing the best posts to comment on daily and warming the network before Mike's posts go live -- the #1 lever for LinkedIn reach.

Create a Supabase migration file at `supabase/migrations/20260215_linkedin_engine_engagement_grants.sql` with:

**1. New `engagement_digests` table:**
```sql
CREATE TABLE linkedin_engine.engagement_digests (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  digest_date DATE NOT NULL,
  digest_type TEXT NOT NULL DEFAULT 'daily', -- 'daily' or 'warming'
  target_post_url TEXT,
  target_post_content TEXT,
  target_author TEXT,
  target_author_url TEXT,
  target_author_followers INT,
  network_contact_id UUID REFERENCES linkedin_engine.engagement_network(id),
  post_engagement JSONB, -- {likes, comments, shares}
  comment_suggestions JSONB, -- [{style: "insight", text: "..."}, {style: "question", text: "..."}]
  status TEXT DEFAULT 'pending', -- pending, completed, skipped
  completed_at TIMESTAMPTZ,
  warming_context TEXT, -- only for warming type: upcoming post topic context
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

**2. Indexes:**
```sql
CREATE INDEX idx_le_digest_date ON linkedin_engine.engagement_digests(digest_date);
CREATE INDEX idx_le_digest_type ON linkedin_engine.engagement_digests(digest_type);
CREATE INDEX idx_le_digest_contact ON linkedin_engine.engagement_digests(network_contact_id);
```

**3. RLS on engagement_digests:**
- Enable RLS
- Service role full access policy
- Authenticated read policy

**4. Anon grants for n8n workflows (CRITICAL -- missing from earlier migrations):**
```sql
GRANT SELECT, INSERT, UPDATE ON linkedin_engine.engagement_network TO anon;
GRANT SELECT, INSERT, UPDATE ON linkedin_engine.comment_activity TO anon;
GRANT SELECT, INSERT, UPDATE ON linkedin_engine.engagement_digests TO anon;
```

**5. Anon RLS policies** for all three tables using the same pattern as `20260214_linkedin_engine_anon_rls.sql`:
```sql
DO $$
DECLARE
  _tables TEXT[] := ARRAY['engagement_network', 'comment_activity', 'engagement_digests'];
  _t TEXT;
BEGIN
  FOREACH _t IN ARRAY _tables LOOP
    EXECUTE format('DROP POLICY IF EXISTS "Anon full access for n8n" ON linkedin_engine.%I', _t);
    EXECUTE format('CREATE POLICY "Anon full access for n8n" ON linkedin_engine.%I FOR ALL TO anon USING (true) WITH CHECK (true)', _t);
  END LOOP;
END $$;
```

**6. Authenticated grants for dashboard CRUD:**
```sql
GRANT INSERT, UPDATE, DELETE ON linkedin_engine.engagement_network TO authenticated;
GRANT UPDATE ON linkedin_engine.engagement_digests TO authenticated;
```

If `supabase db push` fails due to migration history sync issues (known issue from STATE.md), use the Management API workaround: read the SQL file content and execute it via the Supabase SQL editor API or `supabase db execute` directly.

Build a complete n8n workflow JSON file at `n8n-workflows/linkedin-engine/wf6-engagement-engine.json` following the exact JSON structure and patterns from WF3 and WF5. The workflow has TWO independent branches from two schedule triggers.

**BRANCH A: Daily Digest (7 AM CST = 13:00 UTC)**

Nodes in order:
1. **wf6-schedule-daily** (scheduleTrigger v1.2): Cron `0 13 * * *` (7 AM CST daily)
2. **wf6-build-date** (Code v2): Generate `run_id` (UUID v4), `today` (ISO date), `digest_date` for Supabase queries
3. **wf6-log-start** (HTTP Request): POST to `workflow_runs` via Supabase REST with `workflow_name: 'wf6-engagement-engine'`, `trigger_type: 'daily_digest'`, `status: 'running'`
4. **wf6-fetch-network** (HTTP Request): GET `engagement_network?active=eq.true&linkedin_url=not.is.null&select=id,linkedin_name,linkedin_url,linkedin_headline,follower_count,tier,category,last_monitored` from Supabase REST with `Accept-Profile: linkedin_engine`
5. **wf6-check-contacts** (If v2): Check if contacts array is empty. If empty -> skip to log-complete. If not empty -> continue.
6. **wf6-build-profile-urls** (Code v2): Extract `linkedin_url` values from contacts into a `profileUrls` array. Batch into groups of 8 profiles per Apify call (to stay under 300s timeout). Output: array of batches.
7. **wf6-scrape-posts** (HTTP Request): POST to `https://api.apify.com/v2/acts/harvestapi~linkedin-profile-posts/run-sync-get-dataset-items?token={{ $env.APIFY_API_TOKEN }}&timeout=300` with body `{ "profileUrls": <batch>, "maxPosts": 5, "scrapeReactions": false, "scrapeComments": false, "includeReposts": false }`. Use SplitInBatches if multiple batches.
8. **wf6-filter-recent** (Code v2): Filter scraped posts to only those from last 24 hours (compare `postedAt` with current time). Also filter out posts with very low engagement (< 5 reactions). Map each post to include the contact info (match by `publicIdentifier` or profile URL).
9. **wf6-rank-select** (Code v2): Rank filtered posts by engagement score: `(likes * 1) + (comments * 3) + (shares * 2)`. Tier 1 contacts get +10 bonus. Select top 7 posts. If fewer than 5, take all. Output: array of selected posts with contact metadata.
10. **wf6-check-posts** (If v2): Check if any posts were selected. If none -> skip to log-complete with note. If posts -> continue.
11. **wf6-generate-comments** (HTTP Request in SplitInBatches v3): For each selected post, call Claude Sonnet API at `https://api.anthropic.com/v1/messages` with:
    - `anthropic-version: 2023-06-01` header
    - Auth via `httpHeaderAuth` credential (anthropic-api, same as WF3/WF4)
    - Prompt: Generate 2 comment suggestions for the given post. Comment A = insight/value-add style. Comment B = question/discussion style. Follow brand voice: technically curious, observer/builder tone, never prescriber. Subtle IAML references only when directly relevant. Length varies: short for agreement, medium for insights. Output as JSON: `{"suggestions": [{"style": "insight", "text": "..."}, {"style": "question", "text": "..."}]}`
    - Use `max_tokens: 800`
    - Add a Wait 1s node between batches to respect rate limits
12. **wf6-parse-suggestions** (Code v2): Parse Claude JSON responses. Match suggestions back to posts. Build digest item objects.
13. **wf6-store-digest** (HTTP Request): POST array of digest items to `engagement_digests` via Supabase REST with `Content-Profile: linkedin_engine`, `Accept-Profile: linkedin_engine`, `Prefer: return=minimal`. Each item: `{ digest_date, digest_type: 'daily', target_post_url, target_post_content (first 500 chars), target_author, target_author_url, target_author_followers, network_contact_id, post_engagement: {likes, comments, shares}, comment_suggestions, status: 'pending' }`
14. **wf6-update-monitored** (HTTP Request): PATCH `engagement_network` via Supabase REST, set `last_monitored = NOW()` for all contacts that were scraped. Use `id=in.(id1,id2,...)` filter.
15. **wf6-slack-digest** (HTTP Request): POST to Slack webhook `https://hooks.slack.com/services/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` with Block Kit message:
    - Header: "Daily Engagement Digest"
    - Section: "X posts to engage with today"
    - For each post (up to 5): Author name, post snippet (100 chars), engagement stats, 2 comment suggestions
    - Footer: "View full digest in dashboard"
16. **wf6-log-daily-complete** (HTTP Request): PATCH `workflow_runs` via Supabase REST, set `status: 'completed'`, `completed_at: NOW()`, `result_summary: { posts_found, digest_items_created }`

**BRANCH B: Pre-Post Warming (7:40 AM CST = 13:40 UTC, Tue-Fri)**

Nodes in order:
1. **wf6-schedule-warming** (scheduleTrigger v1.2): Cron `40 13 * * 2-5` (7:40 AM CST Tue-Fri)
2. **wf6-warming-date** (Code v2): Generate `run_id`, check day of week (should be Tue-Fri but cron handles this). Build today's date.
3. **wf6-log-warming-start** (HTTP Request): POST to `workflow_runs` with `trigger_type: 'warming'`
4. **wf6-fetch-today-post** (HTTP Request): GET `content_calendar?post_date=eq.<today>&select=id,post_id,series,pillar` then if `post_id` is not null, GET `posts?id=eq.<post_id>&select=id,topic_id,hook_text,full_text,series,pillar` from Supabase REST. This gives the upcoming post's context.
5. **wf6-check-post-exists** (If v2): Check if today has a scheduled/approved post. If no post -> skip warming, log "no post today", done.
6. **wf6-fetch-warming-network** (HTTP Request): GET `engagement_network?active=eq.true&linkedin_url=not.is.null&select=*` from Supabase REST
7. **wf6-select-warming-targets** (Code v2): Score contacts for warming relevance using algorithm from RESEARCH.md:
    - Same category as post topic: +3
    - Tier 1: +2, Tier 2: +1
    - Not engaged in last 48h (check `last_engaged`): +1
    - Select top 4 contacts
8. **wf6-scrape-warming-posts** (HTTP Request): POST to Apify with the 4 selected contact `profileUrls`, `maxPosts: 3`
9. **wf6-filter-warming-posts** (Code v2): Filter to most recent post per contact (last 48h). Select the best post per contact (highest engagement).
10. **wf6-generate-warming-comments** (HTTP Request): Call Claude with post content + upcoming post context. Prompt includes: "Mike is about to publish a post about [topic]. Generate warming comments that seed related conversations. Comment A = value-add, Comment B = question." Output JSON.
11. **wf6-store-warming** (HTTP Request): POST warming digest items to `engagement_digests` with `digest_type: 'warming'`, `warming_context: <upcoming post topic summary>`
12. **wf6-slack-warming** (HTTP Request): POST to Slack webhook with Block Kit message:
    - Header: "Pre-Post Warming Alert"
    - Section: "Publishing in ~20 minutes: [post hook text]"
    - For each warming target: Author, post snippet, 2 comment suggestions
    - CTA: "Engage with these posts NOW to warm the network"
13. **wf6-log-warming-complete** (HTTP Request): PATCH `workflow_runs`, set completed
