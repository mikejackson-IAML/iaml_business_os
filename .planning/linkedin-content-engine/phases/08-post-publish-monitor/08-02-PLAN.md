Build the complete n8n workflow JSON following the exact patterns from WF6. The workflow has a single execution path triggered every 10 minutes.

**WORKFLOW STRUCTURE (sequential flow):**

**Node 1: Schedule Trigger** (`wf7-schedule-10min`)
- Type: `n8n-nodes-base.scheduleTrigger`
- Cron: `*/10 * * * *` (every 10 minutes)
- Position: [0, 300]

**Node 2: Build Run Context** (`wf7-build-context`) - Code node
- Generate run_id via uuidv4()
- Capture started_at timestamp
- Set workflow_name: "wf7-post-publish-monitor"

**Node 3: Log Workflow Start** (`wf7-log-start`)
- POST to `https://mnkuffgxemfyitcjnjdc.supabase.co/rest/v1/workflow_runs`
- Credential: `httpHeaderAuth` / `Dy6aCSbL5Tup4TnE` (predefinedCredentialType)
- Headers: Accept-Profile: linkedin_engine, Content-Profile: linkedin_engine
- Body: `{ id, workflow_name, status: "running", started_at }`
- Prefer: return=minimal

**Node 4: Activate New Published Posts** (`wf7-activate-monitoring`)
- PATCH to `https://mnkuffgxemfyitcjnjdc.supabase.co/rest/v1/posts?status=eq.published&monitoring_status=eq.inactive`
- This finds posts that were just published but not yet being monitored
- Body: `{ "monitoring_status": "active", "monitoring_started_at": "{{ $now.toISO() }}", "monitoring_ends_at": "{{ $now.plus({days: 7}).toISO() }}" }`
- Headers: Accept-Profile: linkedin_engine, Content-Profile: linkedin_engine
- Prefer: return=minimal
- continueOnFail: true (ok if no rows match)

**Node 5: Expire Old Monitoring** (`wf7-expire-monitoring`)
- PATCH to `https://mnkuffgxemfyitcjnjdc.supabase.co/rest/v1/posts?monitoring_status=eq.active&monitoring_ends_at=lt.{{ $now.toISO() }}`
- Body: `{ "monitoring_status": "completed" }`
- Headers: Accept-Profile: linkedin_engine, Content-Profile: linkedin_engine
- Prefer: return=minimal
- continueOnFail: true

**Node 6: Get Active Monitored Posts** (`wf7-get-active-posts`)
- GET `https://mnkuffgxemfyitcjnjdc.supabase.co/rest/v1/posts?monitoring_status=eq.active&select=id,linkedin_post_id,full_text,hook_text,published_at,last_polled_at,monitoring_started_at`
- Headers: Accept-Profile: linkedin_engine
- Credential: httpHeaderAuth / Dy6aCSbL5Tup4TnE

**Node 7: Filter Posts Needing Poll** (`wf7-filter-poll-needed`) - Code node
- Implements the polling schedule logic. For each post:
  - Calculate `hoursSincePublish` from `published_at`
  - Calculate `minutesSinceLastPoll` from `last_polled_at`
  - Apply schedule: <=2h -> every 10min, <=8h -> every 30min, <=24h -> every 60min, <=168h -> every 1440min
  - Filter to only posts whose interval has elapsed
  - Also construct the LinkedIn post URL: `https://www.linkedin.com/feed/update/${linkedin_post_id}/`
- Output: array of posts needing polling (or empty if none need polling right now)
- If output is empty array, return `[{ json: { _skip: true, posts_to_poll: 0 } }]`

**Node 8: Check If Skip** (`wf7-check-skip`) - IF node
- Condition: `{{ $json._skip }}` is NOT true
- True output -> continue to Apify scraping
- False output -> skip to Log Success (no posts need polling)

**Node 9: Loop Over Posts** (`wf7-loop-posts`)
- Type: `n8n-nodes-base.splitInBatches`
- Batch size: 1 (process one post at a time to manage Apify rate)

**Node 10: Scrape Comments via Apify** (`wf7-apify-comments`)
- POST `https://api.apify.com/v2/acts/harvestapi~linkedin-post-comments/run-sync-get-dataset-items?token={{ $env.APIFY_API_TOKEN }}&timeout=120`
- Body: `{ "postUrls": ["{{ $json.linkedin_post_url }}"], "maxItems": 100 }`
- No authentication (token in URL)
- onError: continueRegularOutput (if Apify fails for one post, continue with others)

**Node 11: Get Existing Comment IDs** (`wf7-get-existing-comments`)
- GET `https://mnkuffgxemfyitcjnjdc.supabase.co/rest/v1/post_incoming_comments?post_id=eq.{{ $('wf7-loop-posts').item.json.id }}&select=linkedin_comment_id`
- Headers: Accept-Profile: linkedin_engine
- This fetches existing comment IDs for dedup

**Node 12: Process Comments + Velocity** (`wf7-process-comments`) - Code node
- INPUT: Apify comment results from node 10, existing comment IDs from node 11, post data from loop
- LOGIC:
  1. Extract known comment IDs into a Set for dedup
  2. Filter Apify results to only NEW comments (linkedin_comment_id not in known set)
  3. Map new comments to `post_incoming_comments` row format:
     - `post_id`: from loop item
     - `linkedin_comment_id`: from Apify `id` field
     - `commenter_name`: from Apify `actor.name`
     - `commenter_url`: from Apify `actor.linkedinUrl`
     - `commenter_headline`: from Apify `actor.position` (headline)
     - `comment_text`: from Apify `commentary`
     - `comment_posted_at`: from Apify `createdAt`
     - `comment_likes`: sum of Apify `reactionTypeCounts`
     - `comment_replies_count`: from Apify `numComments`
     - `poll_sequence`: increment counter based on time since publish
  4. Calculate engagement velocity:
     - Total reactions from Apify results (sum all comment likes + reactions)
     - Total comment count
     - hours_since_publish
     - reactions_per_hour, comments_per_hour
  5. Build post_analytics row for this poll:
     - `post_id`, `captured_at: now`, `comments_count: totalComments`, `reactions_total: totalReactions`, `hours_since_publish`
  6. Check for disagreement thread updates:
     - Look for replies to comments where thread_state = 'follow_up_sent'
     - If found, include thread_state update info
- OUTPUT: `{ new_comments: [...], analytics_row: {...}, velocity: {...}, post_data: {...} }`

**Node 13: Check If New Comments** (`wf7-check-new-comments`) - IF node
- Condition: `{{ $json.new_comments.length > 0 }}`
- True -> classify comments
- False -> skip to analytics storage (still store velocity even if no new comments)

**Node 14: Classify Comments via Claude** (`wf7-classify-comments`) - Code node that builds the prompt, followed by HTTP Request
- For EACH new comment, call Claude API:
  - URL: `https://api.anthropic.com/v1/messages`
  - Credential: predefinedCredentialType `httpCustomAuth` / `anthropic-api`
  - Headers: `x-api-key`, `anthropic-version: 2023-06-01`, `content-type: application/json`
  - Model: `claude-sonnet-4-20250514`
  - max_tokens: 800
  - Prompt (from RESEARCH.md):
    ```
    You are helping Mike Van Horn ("The HR Technologist") respond to comments on his LinkedIn posts. Mike is CEO of IAML, a 45-year employment law training company. He bridges genuine AI expertise and deep-domain HR/employment law knowledge.

    REPLY TONE: More conversational than post content. No emojis. Professional but approachable.

    ORIGINAL POST:
    ${postFullText}

    COMMENT by ${commenterName}:
    "${commentText}"

    ${threadContext ? `THREAD CONTEXT: This is a follow-up to a previous disagreement. The user previously disagreed and you asked a follow-up question. Now acknowledge their point and bridge to your perspective.` : ''}

    1. Classify this comment as one of: question, agreement, disagreement, addition, spam
    2. ${isSpam ? 'For spam, return null reply.' : 'Generate a reply suggestion.'}

    Reply length guidelines:
    - agreement: 1 sentence (brief acknowledgment)
    - question: 2-3 sentences (answer with expertise)
    - disagreement: ${threadState === 'follow_up_sent' ? 'Acknowledge their point and bridge to your perspective (2-3 sentences).' : 'Ask a follow-up question (1-2 sentences). Do NOT argue.'}
    - addition: 1-2 sentences (build on their point)

    Respond with ONLY valid JSON:
    {"type": "question|agreement|disagreement|addition|spam", "reply": "...or null for spam"}
    ```

IMPORTANT: Because n8n cannot easily loop HTTP requests inside a Code node, implement this as:
- A Code node (`wf7-prepare-classify`) that prepares an array of items, one per new comment, each with the Claude API request body
- A SplitInBatches node (`wf7-classify-loop`) to iterate
- An HTTP Request node (`wf7-claude-classify`) to call Claude for each comment
- A Code node (`wf7-merge-classified`) that collects all results back together

Alternative simpler approach: If there are typically 1-5 new comments per poll, a single Code node can batch all comments into ONE Claude call with instructions to classify all at once:

```
Classify each comment and generate reply suggestions. Return a JSON array.

ORIGINAL POST:
${postFullText}

COMMENTS:
${comments.map((c, i) => `[${i+1}] by ${c.commenter_name}: "${c.comment_text}"`).join('\n')}

For each comment, return:
{"index": N, "type": "...", "reply": "...or null for spam"}

Return ONLY a JSON array: [{"index": 1, ...}, {"index": 2, ...}]
```

USE THE BATCH APPROACH (one Claude call for all comments). It is simpler and cheaper. Max 10 comments per batch. The HTTP Request node for Claude:
- Method: POST
- URL: `https://api.anthropic.com/v1/messages`
- Authentication: predefinedCredentialType, nodeCredentialType: httpCustomAuth
- Credential ID: `anthropic-api`
- Headers: anthropic-version: 2023-06-01, content-type: application/json
- Body: Built by the preceding Code node

**Node 15: Store Classified Comments** (`wf7-store-comments`)
- POST to `https://mnkuffgxemfyitcjnjdc.supabase.co/rest/v1/post_incoming_comments`
- Headers: Accept-Profile: linkedin_engine, Content-Profile: linkedin_engine, Prefer: return=minimal
- Body: Array of classified comment objects (from merge node)
- Credential: httpHeaderAuth / Dy6aCSbL5Tup4TnE

**Node 16: Store Analytics Row** (`wf7-store-analytics`)
- POST to `https://mnkuffgxemfyitcjnjdc.supabase.co/rest/v1/post_analytics`
- Headers: Accept-Profile: linkedin_engine, Content-Profile: linkedin_engine, Prefer: return=minimal
- Body: The analytics row from process node (post_id, captured_at, comments_count, reactions_total, hours_since_publish, engagement_rate)
- Runs regardless of whether there are new comments (always after node 12)

**Node 17: Update Post Last Polled** (`wf7-update-last-polled`)
- PATCH to `https://mnkuffgxemfyitcjnjdc.supabase.co/rest/v1/posts?id=eq.{{ $json.post_data.id }}`
- Body: `{ "last_polled_at": "{{ $now.toISO() }}" }`
- Headers: Accept-Profile: linkedin_engine, Content-Profile: linkedin_engine

**Node 18: Check Viral Threshold** (`wf7-check-viral`) - Code node
- Calculate if this post is going viral:
  1. Get rolling average first-hour reactions from existing post_analytics (GET request in preceding node or inline)
  2. If fewer than 5 posts have analytics, use fixed fallback: 20 reactions in first hour
  3. Viral = current reactions >= 2x rolling average AND hours_since_publish <= 2
- Output: `{ is_viral: boolean, threshold: number, current: number }`

**Node 19: Viral Slack Alert** (`wf7-slack-viral`) - IF viral -> HTTP Request
- IF node checks `{{ $json.is_viral }}`
- If true: POST to `https://hooks.slack.com/services/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`
- Slack Block Kit message (from RESEARCH.md):

  ```json
        }
      }

  }
  ```

**Node 20: Engagement Boost Trigger** (`wf7-engagement-boost`) - Runs after viral Slack

**Node 21: Priority Comment Notifications** (`wf7-notify-priority`) - Code node + IF + HTTP Request

  ```json
        }
      }
    ]
  }
  ```
- Mark notified=true on these comments via PATCH

**Node 22: Update Disagreement Thread States** (`wf7-update-threads`)
- If any existing disagreements with thread_state='new' had replies detected, update to 'follow_up_sent'
- If any existing disagreements with thread_state='follow_up_sent' had new replies from the same commenter, track for next poll
