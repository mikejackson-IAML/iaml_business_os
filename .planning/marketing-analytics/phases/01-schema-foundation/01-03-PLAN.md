Create a single SQL migration file with 6 RPC functions. All functions follow the same pattern:
- Accept optional parameters (p_campaign_id, p_tier) defaulting to NULL
- Read from the corresponding materialized view
- Filter by tier when p_tier is provided
- Aggregate across tiers when p_tier is NULL (SUM the counts, GROUP BY non-tier columns)
- Use SECURITY DEFINER to bypass RLS on aggregate reads
- Return TABLE type with explicit column definitions

Include a file header comment block.

**Function 1: get_analytics_pipeline**

```sql
CREATE OR REPLACE FUNCTION get_analytics_pipeline(
  p_campaign_id UUID DEFAULT NULL,
  p_tier TEXT DEFAULT NULL
)
RETURNS TABLE (
  campaign_id UUID,
  campaign_name TEXT,
  total_cold BIGINT,
  engaged BIGINT,
  qualified BIGINT,
  registered BIGINT,
  alumni BIGINT
) AS $$
BEGIN
  RETURN QUERY
  SELECT
    pf.campaign_id,
    pf.campaign_name,
    SUM(pf.total_cold)::BIGINT,
    SUM(pf.engaged)::BIGINT,
    SUM(pf.qualified)::BIGINT,
    SUM(pf.registered)::BIGINT,
    SUM(pf.alumni)::BIGINT
  FROM mv_pipeline_funnel pf
  WHERE (p_campaign_id IS NULL OR pf.campaign_id = p_campaign_id)
    AND (p_tier IS NULL OR pf.tier = p_tier)
  GROUP BY pf.campaign_id, pf.campaign_name;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

**Function 2: get_analytics_channels**

Returns per-channel metrics with optional campaign and tier filters. SUM aggregates across tiers when no tier filter. GROUP BY channel_id, campaign_id, channel, platform.

Return columns: channel_id UUID, campaign_id UUID, channel TEXT, platform TEXT, total_contacts BIGINT, sends BIGINT, opens BIGINT, clicks BIGINT, replies BIGINT, bounces BIGINT, connection_requests BIGINT, connections_accepted BIGINT, registrations BIGINT.

Same WHERE pattern: (p_campaign_id IS NULL OR ...) AND (p_tier IS NULL OR ...).

**Function 3: get_analytics_campaigns**

Returns campaign summary cards with optional tier filter. No campaign_id filter (returns all campaigns).

Return columns: campaign_id UUID, campaign_name TEXT, campaign_status TEXT, started_at TIMESTAMPTZ, total_contacts BIGINT, active_contacts BIGINT, replied_contacts BIGINT, registered_contacts BIGINT, branch_a BIGINT, branch_a_plus BIGINT, branch_b BIGINT, branch_c BIGINT, opted_out BIGINT.

GROUP BY campaign_id, campaign_name, campaign_status, started_at.

**Function 4: get_campaign_drilldown**

Returns per-step metrics for a specific campaign. Campaign_id is REQUIRED (not optional -- no default).

```sql
CREATE OR REPLACE FUNCTION get_campaign_drilldown(
  p_campaign_id UUID,
  p_tier TEXT DEFAULT NULL
)
```

Return columns: campaign_id UUID, channel_id UUID, message_id UUID, message_code TEXT, message_name TEXT, message_type TEXT, sequence_order INTEGER, channel TEXT, sends BIGINT, opens BIGINT, clicks BIGINT, replies BIGINT, bounces BIGINT.

GROUP BY all non-aggregate columns. ORDER BY channel, sequence_order.

**Function 5: get_conversion_metrics**

Returns conversion tracking data with optional campaign and tier filters.

Return columns: campaign_id UUID, campaign_name TEXT, total_contacts BIGINT, qu_signups BIGINT, qu_attended BIGINT, secondary_interested BIGINT, secondary_accepted BIGINT, referrals_generated BIGINT, referrals_converted BIGINT, positive_replies BIGINT, total_replies BIGINT.

Same WHERE and GROUP BY pattern.

**Function 6: get_sync_status**

This is different -- it reads from analytics_sync_log directly (not a materialized view). No tier filter.

```sql
CREATE OR REPLACE FUNCTION get_sync_status()
RETURNS TABLE (
  source TEXT,
  last_sync_at TIMESTAMPTZ,
  records_synced INTEGER,
  status TEXT,
  error_message TEXT,
  metadata JSONB
) AS $$
BEGIN
  RETURN QUERY
  SELECT
    s.source,
    s.last_sync_at,
    s.records_synced,
    s.status,
    s.error_message,
    s.metadata
  FROM analytics_sync_log s
  ORDER BY s.source;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

IMPORTANT implementation notes:
- Every function uses `SECURITY DEFINER` to bypass RLS for performance on aggregate reads
- Every function that reads from materialized views uses SUM()::BIGINT to aggregate across tier rows when p_tier IS NULL
- get_campaign_drilldown has p_campaign_id as REQUIRED (no DEFAULT), p_tier as OPTIONAL
- All other functions have both p_campaign_id and p_tier as OPTIONAL with DEFAULT NULL
- Use explicit table aliases (pf, cs, cm, etc.) to avoid column ambiguity with RETURNS TABLE columns
