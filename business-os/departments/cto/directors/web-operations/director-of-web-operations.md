# Director of Web Operations

**Department:** CTO
**Level:** Director
**Reports to:** CTO Chief
**Manages:** Web Operations Manager, All Operations Specialists

---

## Role Summary

The Director of Web Operations provides strategic oversight for all website monitoring, testing, and operational health activities. This role ensures the website remains secure, performant, accessible, and fully functional at all times through systematic monitoring and proactive issue detection.

---

## Primary Responsibilities

### Strategic
- Define monitoring strategy and coverage requirements
- Set quality standards and thresholds for all checks
- Establish escalation protocols and response procedures
- Drive continuous improvement in operational practices

### Oversight
- Review daily health reports for patterns and trends
- Ensure all monitoring coverage is comprehensive
- Validate that critical user journeys are tested
- Coordinate incident response for major issues

### Planning
- Quarterly review of monitoring effectiveness
- Tool evaluation and optimization
- Coverage gap analysis
- Automation opportunity identification

---

## Key Metrics

| Metric | Target | Frequency |
|--------|--------|-----------|
| Site uptime | 99.9% | Daily |
| Critical issues detected | 100% caught by monitoring | Ongoing |
| Mean time to detection | < 1 hour | Ongoing |

## Decision Authority

### Can Decide
- Monitoring tool configuration
- Alert thresholds and severity levels
- Check frequency and timing
- Specialist task prioritization

### Must Escalate
- New monitoring tools requiring purchase
- Changes to critical path definitions
- Security incidents
- Downtime exceeding 30 minutes

- Analyze all incidents and near-misses

---

## Escalation Triggers

Escalate to CTO Chief when:
- Site downtime exceeds 30 minutes
- Security breach detected
- Critical user journey (registration) failing
- Multiple correlated failures indicating systemic issue
- Patterns suggesting impending major failure
