# Opportunities Registry

> Improvement ideas, optimizations, and growth opportunities to pursue.

---

## Purpose

This registry captures:
1. Ideas for doing things better (not fixing problems, but improving)
2. Growth opportunities spotted
3. Automation candidates
4. Efficiency improvements

The Chief Improvement Officer reviews these to prioritize experiments.

---

## How to Log Opportunities

When you spot an opportunity, add it here:

```markdown
### [Brief Title]

**Category:** [Efficiency/Growth/Automation/Quality/Experience]
**Potential Impact:** 🔴 High / 🟠 Medium / 🟡 Low
**Effort Required:** 🔴 High / 🟠 Medium / 🟡 Low
**Logged:** [Date]
**Status:** Idea / Evaluating / Approved / In Progress / Implemented

**Opportunity:**
[Clear description of the improvement]

**Expected Benefit:**
[What we'd gain - time saved, revenue increase, quality improvement]

**How to Implement:**
[High-level approach]

**Resources Needed:**
- [Tools/time/skills required]

**Success Metrics:**
[How we'd measure if this worked]

**Notes:**
[Any additional context]
```

## High-Priority Opportunities

### Automated Daily Business Pulse

**Category:** Efficiency
**Potential Impact:** 🔴 High
**Effort Required:** 🟠 Medium
**Logged:** 2024-XX-XX
**Status:** Approved

**Opportunity:**
Create automated daily briefing that runs each morning, pulling data from all sources and presenting unified view.

**Expected Benefit:**
- Save 30+ min/day checking multiple systems
- Never miss important changes
- Start each day informed and focused

**How to Implement:**
1. Create daily briefing skill
2. Connect all MCP servers
3. Define key metrics to track
4. Set up morning trigger

**Resources Needed:**
- SmartLead, Supabase, Airtable MCP servers
- Daily briefing skill development
- ~4 hours initial setup

**Success Metrics:**
- Briefing runs automatically each morning
- Contains all key metrics
- Time saved: 30 min/day
