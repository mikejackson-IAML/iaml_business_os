## Detailed Findings

### 1. Images

**Status: NEEDS ATTENTION**

- `/website/images/` contains only 2 SVG files (`iaml-logo.svg`, `name-underline.svg`) — no raster images locally
- **No WebP or AVIF images** found anywhere in the project
- All photos are loaded from external CDNs: `storage.googleapis.com`, `i.imgur.com`, `images.unsplash.com`
- External images are served as JPEG/SVG — no format optimization possible without hosting locally
- **corporate-training.html**: 10 company logo `<img>` tags (lines 1333–1540) are missing:
  - `loading="lazy"` attribute
  - `width` and `height` attributes (CLS risk)
- **index.html**: 9 `<img>` tags use JS-driven lazy loading (`data-src` pattern) — acceptable but depends on JS being available

**Recommendation**: 
- Add `loading="lazy"` and explicit `width`/`height` to all non-above-fold images
- Add `width`/`height` to all `<img>` tags to prevent CLS
- Consider hosting key images locally and serving as WebP with JPEG fallback

### 2. CSS Efficiency

**Status: ATTENTION NEEDED**

| File | Size | Notes |
|------|------|-------|
| main.css | **292 KB** | Primary stylesheet — very large for a static site |
| mega-menu.css | 35 KB | Loaded on all pages |
| 5-pages.css | 225 KB | Reference only (not loaded by pages directly) |
| 3-components.css | 49 KB | Reference only |

- `main.css` at 292 KB unminified is the primary performance concern
- All pages load both `main.css` and `mega-menu.css` regardless of page-specific needs
- **1,631 CSS rule selectors** in main.css — significant potential for unused rules on any given page
- No CSS minification in the build pipeline
- Font Awesome 5 (`all.min.css`) loaded **only on corporate-training.html** (line 41) — adds ~200 KB of CSS for icons that could be replaced with SVGs
- Fontshare CSS is loaded synchronously via `<link rel="stylesheet">` in `<head>` — this is render-blocking (though `preconnect` hints help mitigate)

**Recommendations**:
- Minify main.css (could reduce to ~180–200 KB)
- Replace Font Awesome on corporate-training.html with inline SVG icons (saves ~200 KB CSS)
- Audit main.css for dead rules (consider PurgeCSS analysis)
- Consider critical CSS inlining for above-fold styles

### 3. JavaScript — Render-Blocking Scripts

**Status: TWO CRITICAL ISSUES**

#### ✅ Good patterns

- GA4 (`gtag.js`) uses `async` attribute

**corporate-training.html, line 1614**:

- Missing `async` or `defer` — **render-blocking**

**register.html, line 31**:

### 4. Font Loading

**Status: ACCEPTABLE with caveats**

- Fontshare fonts are loaded via: `https://api.fontshare.com/v2/css?...&display=swap`
- The `display=swap` URL parameter instructs Fontshare to include `font-display: swap` in their CSS response ✅
- `preconnect` to `api.fontshare.com` and `cdn.fontshare.com` is set on index.html and corporate-training.html ✅
- **contact-us.html** has `preconnect` to Fontshare but NOT to `cdn.jsdelivr.net`, `storage.googleapis.com`, or `www.googletagmanager.com` ❌
- No `<link rel="preload">` for font files — fonts are discovered only after the CSS is fetched and parsed
- No `@font-face` declarations in local CSS — fully relying on Fontshare serving `font-display: swap`

**Recommendations**:
- Add `preconnect` to all relevant third-party domains on all pages (contact-us.html is missing several)
- Consider self-hosting fonts to eliminate the Fontshare network dependency

### 5. Caching

**Status: INCOMPLETE**

From `vercel.json`, Cache-Control headers are only set for:
- `/data/sessions/*` — 1 hour + SWR
- `/data/faculty/*` — 1 hour + SWR
- `/data/testimonials/*` — 1 hour + SWR
- `/data/organizations/*` — 1 hour + SWR

**Missing cache headers for**:
- `/css/*.css` — static assets should have long TTL (e.g., `max-age=31536000, immutable`)
- `/js/*.js` — same
- `/images/*` — same
- `*.html` — should have short TTL with `stale-while-revalidate`
- `/favicon.ico`, `/apple-touch-icon.png` — should be cached long-term

Vercel's default for static assets varies but adding explicit headers ensures consistent behavior.

**Recommendation**: Add cache headers to vercel.json for all static asset types.
