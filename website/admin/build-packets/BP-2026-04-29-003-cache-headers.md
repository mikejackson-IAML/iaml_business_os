# Build Packet BP-2026-04-29-003
**Title**: Add Cache-Control Headers for Static Assets in vercel.json  
**Priority**: MEDIUM  
**Source**: Performance Audit 2026-04-29  
**Estimated Effort**: 15 minutes

---

## Problem

`vercel.json` currently only sets Cache-Control headers for `/data/*` API routes. Static assets (CSS, JS, images, fonts) have no explicit cache configuration.

Without explicit headers, Vercel may serve these with `Cache-Control: public, max-age=0, must-revalidate` or similar short-lived policies, causing unnecessary re-downloads for repeat visitors.

---

## Changes Required

### File: `website/vercel.json`

Add the following header blocks to the `"headers"` array:

```json
{
  "source": "/css/(.*)",
  "headers": [
    {
      "key": "Cache-Control",
      "value": "public, max-age=31536000, immutable"
    }
  ]
},
{
  "source": "/js/(.*)",
  "headers": [
    {
      "key": "Cache-Control",
      "value": "public, max-age=31536000, immutable"
    }
  ]
},
{
  "source": "/images/(.*)",
  "headers": [
    {
      "key": "Cache-Control",
      "value": "public, max-age=31536000, immutable"
    }
  ]
},
{
  "source": "/(.*)\\.ico",
  "headers": [
    {
      "key": "Cache-Control",
      "value": "public, max-age=31536000, immutable"
    }
  ]
},
{
  "source": "/(.*)\\.png",
  "headers": [
    {
      "key": "Cache-Control",
      "value": "public, max-age=31536000, immutable"
    }
  ]
},
{
  "source": "/(.*)\\.html",
  "headers": [
    {
      "key": "Cache-Control",
      "value": "public, max-age=0, must-revalidate"
    }
  ]
}
```

**Important**: `max-age=31536000, immutable` (1 year) is appropriate for CSS/JS **only if** filenames include a content hash or version string for cache-busting. Since IAML uses plain filenames (e.g., `main.css`, `components.js`), a shorter TTL like `max-age=86400` (1 day) or `max-age=604800` (1 week) with `stale-while-revalidate` may be safer to allow updates to propagate.

**Recommended safer approach for non-versioned files**:
```json
"value": "public, max-age=86400, stale-while-revalidate=604800"
```

This caches for 1 day, serves stale for up to 7 days while revalidating in the background.

## QA Steps

1. Deploy to Vercel preview
2. Open DevTools → Network tab → check `Cache-Control` response headers on `main.css` and `components.js`
3. Reload page — verify assets served from cache (304 or `from disk cache`)
4. Run `/deployed-smoke` to verify all pages load correctly
5. After a CSS update: verify the updated CSS loads (not stale) — key test for cache-busting safety
