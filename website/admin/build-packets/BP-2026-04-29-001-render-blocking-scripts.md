## Problem

Two third-party scripts are loaded synchronously without `async` or `defer`, blocking HTML parsing and delaying page render:

1. **corporate-training.html, line 1614**: GoHighLevel/msgsndr form embed script
2. **register.html, lines 31–32**: Stripe.js and stripe-products.js

These cause render-blocking that can delay LCP by 300–800ms on slow connections.

---

## Changes Required

### File 1: `website/corporate-training.html`

**Line 1614** — Change:
```html
<script src="https://link.msgsndr.com/js/form_embed.js" type="text/javascript"></script>
```
To:
```html
<script src="https://link.msgsndr.com/js/form_embed.js" defer></script>
```

**Note**: Test that the embedded form still renders correctly after deferring. The form embed likely targets a `<div>` placeholder — it should work deferred as long as it runs after DOMContentLoaded.

### File 2: `website/register.html`

**Lines 31–32** — Change:
```html
<script src="https://js.stripe.com/v3/"></script>
<script src="js/stripe-products.js"></script>
```
To:
```html
<script src="https://js.stripe.com/v3/" async></script>
<script src="js/stripe-products.js" defer></script>
```

**Note**: If `stripe-products.js` initializes Stripe synchronously at load time (i.e., calls `Stripe(...)` at top level), changing to `async` could cause a race condition where `stripe-products.js` runs before the Stripe library is ready. Verify that `stripe-products.js` waits for the Stripe library to be available (e.g., checks `window.Stripe`) before initializing.
