## Problem

`corporate-training.html` has 10 company logo `<img>` tags (lines 1333–1540) that:

1. **No `loading="lazy"`** — all images load eagerly, consuming bandwidth upfront
2. **No `width` or `height` attributes** — browser can't reserve space before image loads, causing Cumulative Layout Shift (CLS)

CLS > 0.1 is a Core Web Vitals failure and hurts Google Search ranking.

---

**Example transformation**:

Before:
```html
<img src="https://storage.googleapis.com/msgsndr/.../media/68fe5c80a4758a0d21515983.svg" alt="Walmart logo" class="company-logo">
```

After:
```html
<img src="https://storage.googleapis.com/msgsndr/.../media/68fe5c80a4758a0d21515983.svg" alt="Walmart logo" class="company-logo" loading="lazy" width="120" height="60">
```

**Note on dimensions**: Adjust `width`/`height` values to match the actual rendered size of `.company-logo` in CSS. Check `main.css` for `.company-logo` dimensions first. The goal is the correct aspect ratio to prevent layout shift, not pixel-perfect matching.
