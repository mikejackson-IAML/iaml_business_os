## Objective

Document the alert refresh strategy and create README documentation for the periodic n8n workflow design.

## Context

The research identified two options:
1. Trigger-based: Database triggers on claims/tier changes auto-refresh
2. Periodic n8n: Workflow calls refresh_alerts() every 15 minutes

Given that:
- Alerts are computed on-the-fly via the `refresh_alerts()` function
- Dashboard queries already call `refresh_alerts()` before fetching alerts (Plan 08-02)
- Most alert conditions are time-based (tier ending, days since notification)

**Decision:** Dashboard query handles on-demand refresh. A periodic n8n workflow provides belt-and-suspenders coverage for alerts that become true over time without user interaction.
