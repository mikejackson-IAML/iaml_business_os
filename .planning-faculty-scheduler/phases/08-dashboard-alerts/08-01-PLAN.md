## Objective

Create the alerts table, refresh_alerts() function, active_alerts view, and configuration entries for the Faculty Scheduler dashboard alerts feature.

## Context

This migration builds on:
- `scheduled_programs` table with tier timing fields (`tier_0_ends_at`, `tier_1_ends_at`)
- `notifications` table with `viewed_at` tracking (Phase 6)
- `faculty` table with `tier_designation` (0 = VIP)
- `n8n_brain.preferences` table for configuration storage

## must_haves

- [ ] `faculty_scheduler.alerts` table exists with alert_type, severity, status, dismiss tracking
- [ ] Unique constraint prevents duplicate alerts per program/instructor
- [ ] `refresh_alerts()` function creates tier_ending and vip_non_response alerts
- [ ] `refresh_alerts()` auto-resolves alerts when conditions change
- [ ] `active_alerts` view returns only active, non-dismissed alerts
- [ ] Configuration entries exist in n8n_brain.preferences for threshold values

<task id="2">
Create `faculty_scheduler.alerts` table with all required columns:
- `id` UUID PRIMARY KEY DEFAULT gen_random_uuid()
- `alert_type` TEXT NOT NULL CHECK (alert_type IN ('tier_ending', 'vip_non_response'))
- `scheduled_program_id` UUID REFERENCES faculty_scheduler.scheduled_programs(id) ON DELETE CASCADE
- `instructor_id` UUID REFERENCES faculty(id) ON DELETE CASCADE (for VIP alerts, nullable)
- `notification_id` UUID REFERENCES faculty_scheduler.notifications(id) (for VIP alerts, nullable)
- `status` TEXT DEFAULT 'active' CHECK (status IN ('active', 'dismissed', 'resolved'))
- `severity` TEXT NOT NULL CHECK (severity IN ('warning', 'critical'))
- `title` TEXT NOT NULL (denormalized for display)
- `description` TEXT
- `triggered_at` TIMESTAMPTZ DEFAULT NOW()
- `dismissed_at` TIMESTAMPTZ
- `dismissed_by` TEXT
- `resolved_at` TIMESTAMPTZ
- UNIQUE(alert_type, scheduled_program_id, COALESCE(instructor_id, '00000000-0000-0000-0000-000000000000'))
</task>

<task id="3">
Create helper function `faculty_scheduler.get_alert_threshold(p_key TEXT)` that reads from n8n_brain.preferences with defaults:
```sql
CREATE OR REPLACE FUNCTION faculty_scheduler.get_alert_threshold(p_key TEXT)
RETURNS INTEGER AS $$
  SELECT COALESCE(
    (SELECT (value)::INTEGER FROM n8n_brain.preferences
     WHERE category = 'faculty_scheduler' AND key = p_key),
    CASE p_key
      WHEN 'tier_ending_alert_hours' THEN 24
      WHEN 'vip_non_response_days' THEN 3
      ELSE 24
    END
  );
$$ LANGUAGE SQL STABLE;
```
</task>

<task id="4">
Create `faculty_scheduler.refresh_alerts()` function that:
1. Inserts new tier_ending alerts for programs with open blocks where current tier ends within threshold
   - severity = 'critical'
   - title format: "{Program Name} - {Tier} ends in {X} hours"
   - description format: "{N} open blocks, {M} instructors notified"
2. Inserts new vip_non_response alerts for VIP instructors who haven't viewed after N days
   - severity = 'warning'
   - title format: "{Instructor Name} hasn't viewed {Program Name}"
   - description format: "VIP instructor notified {N} days ago"
3. Auto-resolves tier_ending alerts when all blocks claimed OR tier advanced
4. Auto-resolves vip_non_response alerts when instructor views (viewed_at set) or claims
5. Returns counts: created_count, resolved_count, active_count
</task>

<task id="5">
Create `faculty_scheduler.active_alerts` view:
```sql
CREATE OR REPLACE VIEW faculty_scheduler.active_alerts AS
SELECT
  id,
  alert_type,
  scheduled_program_id,
  instructor_id,
  severity,
  title,
  description,
  triggered_at
FROM faculty_scheduler.alerts
WHERE status = 'active'
ORDER BY
  severity = 'critical' DESC,
  triggered_at DESC;
```
</task>

<task id="6">
Create `faculty_scheduler.dismiss_alert(p_alert_id UUID, p_dismissed_by TEXT)` function:
```sql
CREATE OR REPLACE FUNCTION faculty_scheduler.dismiss_alert(
  p_alert_id UUID,
  p_dismissed_by TEXT DEFAULT 'dashboard'
)
RETURNS VOID AS $$
BEGIN
  UPDATE faculty_scheduler.alerts
  SET
    status = 'dismissed',
    dismissed_at = NOW(),
    dismissed_by = p_dismissed_by
  WHERE id = p_alert_id
    AND status = 'active';
END;
$$ LANGUAGE plpgsql;
```
</task>
