# 04-01: Supabase Admin Functions

## Objective
Create database functions for bulk release, tier skipping, and notification helpers for reminders and re-release workflows.

## Tasks

<task id="1">
Create migration file `supabase/migrations/20260121_faculty_scheduler_phase4.sql` with header comment explaining Phase 4 purpose.
</task>

<task id="2">
Add 'rerelease' to the notification_type CHECK constraint in faculty_scheduler.notifications table:
- Use ALTER TABLE to drop and recreate the constraint
- New valid types: 'tier_release', 'reminder', 'claim_confirmation', 'claim_cancelled', 'program_update', 'rerelease'
</task>

<task id="3">
Create `faculty_scheduler.release_all()` function:
- Parameters: p_tier_0_days INTEGER DEFAULT 7, p_tier_1_days INTEGER DEFAULT 5
- Returns: TABLE (programs_released INTEGER, program_ids UUID[])
- Logic: Select all programs with status='draft', call release_program logic for each
- Transaction: Single UPDATE statement with RETURNING to capture IDs
</task>

<task id="4">
Create `faculty_scheduler.skip_tier()` function:
- Parameters: p_program_id UUID, p_target_tier TEXT (must be 'tier_1' or 'tier_2')
- Returns: TABLE (success BOOLEAN, previous_tier TEXT, new_tier TEXT, error_message TEXT)
- Validation: Program must exist, must not be 'draft', 'filled', or 'completed'
- Logic: Update status to target tier, adjust tier end dates if needed
- Safety: Return error if program already at or past target tier
</task>

<task id="5">
Create `faculty_scheduler.get_programs_needing_reminder()` function:
- Parameters: none
- Returns: TABLE (program_id UUID, program_name TEXT, city TEXT, state TEXT, current_tier INTEGER, tier_start TIMESTAMPTZ, tier_ends TIMESTAMPTZ, percent_elapsed NUMERIC)
- Logic: For tier_0 and tier_1 programs, calculate % elapsed in current tier window
  - tier_start for tier_0: `released_at`
  - tier_start for tier_1: `tier_0_ends_at`
  - tier_ends for tier_0: `tier_0_ends_at`
  - tier_ends for tier_1: `tier_1_ends_at`
  - percent_elapsed: `EXTRACT(EPOCH FROM (NOW() - tier_start)) / EXTRACT(EPOCH FROM (tier_ends - tier_start)) * 100`
- Filter: Return only programs where percent_elapsed BETWEEN 45 AND 55 (around 50%)
- Exclude: Programs with no open blocks (join to program_blocks WHERE status='open')
- Exclude: Programs that have already had a reminder sent (check notifications table for type='reminder')
</task>

<task id="6">
Create `faculty_scheduler.get_instructors_needing_reminder()` function:
- Parameters: p_scheduled_program_id UUID
- Returns: TABLE (instructor_id UUID, email TEXT, first_name TEXT, last_name TEXT, magic_token TEXT, open_block_count INTEGER)
- Logic: Get eligible instructors at current tier who haven't received 'reminder' notification for this program
- Open block count SQL: `(SELECT COUNT(*) FROM faculty_scheduler.program_blocks WHERE scheduled_program_id = p_scheduled_program_id AND status = 'open')::INTEGER`
- Use: get_or_create_magic_token(f.id) for magic link
- Exclude: Instructors who already have notification with type='reminder' for this program_id
</task>

<task id="7">
Create `faculty_scheduler.get_instructors_for_rerelease()` function:
- Parameters: p_scheduled_program_id UUID, p_block_id UUID DEFAULT NULL
- Returns: TABLE (instructor_id UUID, email TEXT, first_name TEXT, last_name TEXT, magic_token TEXT, program_name TEXT, city TEXT, state TEXT)
- Logic: Get eligible instructors at current tier for notification
- Exclude: Instructors who already have an active claim on any block in this program (JOIN claims WHERE status='confirmed')
- Use: get_or_create_magic_token(f.id) for magic link
- Note: p_block_id is optional, used for filtering but notification goes to all eligible instructors
- Note: block_name is passed separately via webhook payload, not queried here (simpler)
</task>
