# 04-03: Cancellation Re-release Workflow

## Objective
Create n8n workflow triggered by webhook when a claim is cancelled, sending immediate notifications to all qualified instructors that a spot has opened up.

## Tasks

<task id="1">
Create new n8n workflow "Faculty Scheduler - Cancellation Re-release" with:
- Webhook trigger
- Path: /webhook/faculty-scheduler-rerelease
- HTTP Method: POST
- Response Mode: responseNode (to confirm receipt to caller)
- Description: "Notifies qualified instructors when a claim is cancelled and spot reopens"
</task>

<task id="2">
Add Set node "Extract Webhook Data":
- program_id: `{{ $json.body.program_id }}`
- block_id: `{{ $json.body.block_id }}`
- block_name: `{{ $json.body.block_name }}`
- cancelled_by: `{{ $json.body.cancelled_by }}`

Note: Webhook body uses `$json.body.field` pattern (n8n-brain learning #1)
</task>

<task id="3">
Add Postgres node "Get Program Details":
- Query: `SELECT
    sp.id, sp.name, sp.city, sp.state, sp.status, sp.start_date,
    CASE sp.status
      WHEN 'tier_0' THEN 0
      WHEN 'tier_1' THEN 1
      WHEN 'tier_2' THEN 2
    END as current_tier
  FROM faculty_scheduler.scheduled_programs sp
  WHERE sp.id = '{{ $json.program_id }}'::UUID`
- Enable "Always Output Data" = true
</task>

<task id="4">
Add If node "Program Active":
- Condition: `{{ $json.status }}` is one of ['tier_0', 'tier_1', 'tier_2']
- TypeValidation: loose
- False path: Respond with "Program not in active tier, no notifications sent"
</task>

<task id="5">
Add Postgres node "Get Instructors for Re-release":
- Query: `SELECT * FROM faculty_scheduler.get_instructors_for_rerelease('{{ $('Get Program Details').item.json.id }}')`
- Enable "Always Output Data" = true
</task>

<task id="6">
Add If node "Has Instructors":
- Condition: `{{ $json.instructor_id }}` exists
- TypeValidation: loose
- False path: Respond with "No eligible instructors to notify"
</task>

<task id="7">
Add Set node "Count Instructors":
- instructor_count: `{{ $input.all().length }}`
</task>

<task id="8">
Add Loop Over Items node "For Each Instructor":
- Batch size: 1 (send one email at a time)
</task>

<task id="9">
Add SendGrid node "Send Re-release Email":
- To: `{{ $json.email }}`
- From: faculty@iaml.com
- Subject: `New Opening: {{ $('Get Program Details').item.json.name }} - {{ $('Extract Webhook Data').item.json.block_name }}`
- Body (HTML):
  - Greeting: "Hi {{ $json.first_name }},"
  - Content: "Great news! A teaching spot has just opened up."
  - Program: "[program_name] in [city], [state]"
  - Block: "[block_name]"
  - Date: "[start_date]"
  - Urgency: "This spot was just released and is available on a first-come, first-served basis."
  - CTA button: "Claim This Spot" linking to `https://faculty.iaml.com/schedule?token={{ $json.magic_token }}`
</task>

<task id="10">
Add Postgres node "Log Notification":
- Query: `SELECT faculty_scheduler.log_notification(
    '{{ $json.instructor_id }}'::UUID,
    '{{ $('Get Program Details').item.json.id }}'::UUID,
    'rerelease',
    {{ $('Get Program Details').item.json.current_tier }},
    '{{ $json.email }}',
    'New Opening: {{ $('Get Program Details').item.json.name.replace(/'/g, "''") }}',
    'sent',
    NULL,
    '{"block_name": "{{ $('Extract Webhook Data').item.json.block_name.replace(/'/g, "''") }}"}'::JSONB
  )`

Note: SQL apostrophes escaped with `.replace(/'/g, "''")` (n8n-brain learning #5)
</task>

<task id="11">
Add Respond to Webhook node "Success Response":
- Response Code: 200
- Response Body: `{ "success": true, "instructors_notified": {{ $('Count Instructors').item.json.instructor_count }} }`
</task>

<task id="12">
Add standard Canary error handling pattern:

Document webhook integration in workflow notes:

- No automatic database trigger - webhook is called explicitly by admin action

## Verification
- [ ] Webhook accepts POST with program_id, block_id, block_name
- [ ] Program status checked before sending notifications
- [ ] get_instructors_for_rerelease() excludes instructors with active claims
- [ ] Email sent via SendGrid with urgent messaging
- [ ] Notification logged to database with type='rerelease'
- [ ] Webhook responds with count of instructors notified
- [ ] Canary error handling logs errors

## must_haves
- Webhook can be called from admin dashboard (Phase 5 integration)
- Only instructors eligible at current tier receive notification
- Instructors with existing claims on this program are excluded
- Notification logged with type='rerelease' for audit trail
- Response confirms how many instructors were notified

## Integration Notes
The webhook is designed to be called explicitly by the admin dashboard (Phase 5) after a successful cancel_claim() call. This is intentional:
1. Not all cancellations may warrant re-notification (e.g., program cancelled entirely)
2. Admin has control over when notifications go out
3. Simpler implementation - no database triggers needed
