# 04-02: Reminder Workflow

## Objective
Create n8n workflow that sends reminder emails to instructors when programs are at 50% through their current tier window.

## Tasks

<task id="1">
Create new n8n workflow "Faculty Scheduler - Reminder Notifications" with:
- Schedule trigger: Daily at 7:00 AM CT
- Description: "Sends reminder emails at 50% through tier window"
</task>

<task id="2">
Add Postgres node "Get Programs Needing Reminder":
- Credential: Supabase Postgres (EgmvZHbvINHsh6PR)
- Operation: Execute Query
- Query: `SELECT * FROM faculty_scheduler.get_programs_needing_reminder()`
- Enable "Always Output Data" = true
</task>

<task id="3">
Add If node "Has Programs":
- Condition: `{{ $json.program_id }}` exists
- TypeValidation: loose
- Route: true continues, false goes to "No Programs" node
</task>

<task id="4">
Add "No Programs" NoOp node for clean flow termination when no reminders needed.
</task>

<task id="5">
Add Loop Over Items node "For Each Program":
- Batch size: 1 (process one program at a time)
</task>

<task id="6">
Add Postgres node "Get Instructors to Remind":
- Query: `SELECT * FROM faculty_scheduler.get_instructors_needing_reminder('{{ $json.program_id }}')`
- Enable "Always Output Data" = true
</task>

<task id="7">
Add If node "Has Instructors":
- Condition: `{{ $json.instructor_id }}` exists
- TypeValidation: loose
</task>

<task id="8">
Add Loop Over Items node "For Each Instructor":
- Batch size: 1 (send one email at a time to avoid rate limits)
</task>

<task id="9">
Add SendGrid node "Send Reminder Email":
- To: `{{ $json.email }}`
- From: faculty@iaml.com
- Subject: `Reminder: {{ $json.open_block_count }} teaching opportunity{{ $json.open_block_count > 1 ? 's' : '' }} still available`
- Body (HTML):
  - Greeting: "Hi {{ $json.first_name }},"
  - Content: "This is a friendly reminder that teaching opportunities for {{ $('Get Programs Needing Reminder').item.json.program_name }} in {{ $('Get Programs Needing Reminder').item.json.city }}, {{ $('Get Programs Needing Reminder').item.json.state }} are still available."
  - Block count: "There are {{ $json.open_block_count }} blocks still open for this program."
  - CTA button: "View Available Programs" linking to `https://faculty.iaml.com/schedule?token={{ $json.magic_token }}`
  - Note about tier window closing soon

Note: `$json.open_block_count` comes from the instructor query result (get_instructors_needing_reminder).
Program details accessed via `$('Get Programs Needing Reminder').item.json.*` from the outer loop.
</task>

<task id="10">
Add Postgres node "Log Notification":
- Query: `SELECT faculty_scheduler.log_notification(
    '{{ $json.instructor_id }}'::UUID,
    '{{ $('Get Programs Needing Reminder').item.json.program_id }}'::UUID,
    'reminder',
    {{ $('Get Programs Needing Reminder').item.json.current_tier }},
    '{{ $json.email }}',
    'Reminder: Teaching opportunity still available',
    'sent'
  )`
</task>

<task id="11">
Add standard Canary error handling pattern:
- Error Trigger node → Set Error Context → Postgres Log Error → If Should Alert → Slack Notification
- Reference: business-os/docs/standards/WORKFLOW-STANDARDS.md
</task>

<task id="12">
Test workflow manually:
- Create a test program in tier_0 with released_at set to ~3.5 days ago
- Verify get_programs_needing_reminder() returns the program
- Verify email is sent and notification is logged
</task>

## Verification
- [ ] Workflow triggers daily at 7 AM CT
- [ ] get_programs_needing_reminder() correctly identifies programs at ~50% tier window
- [ ] get_programs_needing_reminder() excludes programs that already had a reminder sent
- [ ] get_instructors_needing_reminder() excludes already-reminded instructors
- [ ] Email sent via SendGrid with magic link
- [ ] Notification logged to database with type='reminder'
- [ ] Running workflow twice in same day does NOT send duplicate reminders
- [ ] Canary error handling logs errors to n8n_brain.workflow_errors
- [ ] No emails sent when no programs need reminders

## must_haves
- Reminder email sent at approximately 50% through tier window
- Only instructors who haven't received a reminder for this program get emailed
- Notification logged with type='reminder' for audit trail
- Standard error handling prevents silent failures
