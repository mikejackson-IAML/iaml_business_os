# Plan 07-01: Database Migration for Teaching History

## Objective

Create the `teaching_history` table and supporting triggers/functions to automatically capture instructor teaching records when claims are made.

## Why This Matters

Teaching history enables IAML to make better assignment decisions by showing which programs each instructor has taught before.

## must_haves

- [ ] `faculty_scheduler.teaching_history` table exists with required columns (instructor_id, scheduled_program_id, program_name, program_type, city, state, start_date, end_date, block_count, status, claimed_at, completed_at, cancelled_at)
- [ ] Unique constraint on (instructor_id, scheduled_program_id) ensures one history record per instructor per program
- [ ] Trigger fires on claims INSERT to create/update history record
- [ ] Trigger fires on claims UPDATE (status='cancelled') to update history record
- [ ] Function exists to auto-complete history records where end_date has passed

## Tasks

<task id="1">
Create migration file `supabase/migrations/20260122_faculty_scheduler_phase7_history.sql` with:

1. **teaching_history table** in `faculty_scheduler` schema:
   - `id` UUID PRIMARY KEY DEFAULT gen_random_uuid()
   - `instructor_id` UUID NOT NULL REFERENCES faculty(id) ON DELETE CASCADE
   - `scheduled_program_id` UUID NOT NULL REFERENCES faculty_scheduler.scheduled_programs(id) ON DELETE CASCADE
   - `program_name` TEXT NOT NULL (denormalized for historical accuracy)
   - `program_type` TEXT
   - `city` TEXT
   - `state` TEXT
   - `start_date` DATE NOT NULL
   - `end_date` DATE
   - `block_count` INTEGER DEFAULT 1
   - `status` TEXT DEFAULT 'pending' CHECK (status IN ('pending', 'completed', 'cancelled'))
   - `claimed_at` TIMESTAMPTZ NOT NULL
   - `completed_at` TIMESTAMPTZ
   - `cancelled_at` TIMESTAMPTZ
   - `created_at` TIMESTAMPTZ DEFAULT NOW()
   - `updated_at` TIMESTAMPTZ DEFAULT NOW()
   - UNIQUE(instructor_id, scheduled_program_id)

2. **Indexes** for common queries:
   - `idx_teaching_history_instructor` on (instructor_id)
   - `idx_teaching_history_program` on (scheduled_program_id)
   - `idx_teaching_history_status` on (status)
</task>

<task id="2">
Create trigger function `faculty_scheduler.create_or_update_teaching_history()`:

```sql
-- Called on claims INSERT
-- 1. Look up program details from scheduled_programs via program_blocks
-- 2. Check if history record exists for this instructor/program
-- 3. If exists: INCREMENT block_count
-- 4. If not exists: INSERT new history record with denormalized program info
```

The function should:
- Join claims -> program_blocks -> scheduled_programs to get program details
- Use INSERT ... ON CONFLICT (instructor_id, scheduled_program_id) DO UPDATE to handle both cases
- Set `claimed_at` to the first claim timestamp (use LEAST for ON CONFLICT)
</task>

<task id="3">
Create trigger `trg_claims_insert_history` on `faculty_scheduler.claims`:
- AFTER INSERT
- FOR EACH ROW
- WHEN (NEW.status = 'confirmed')
- EXECUTE FUNCTION faculty_scheduler.create_or_update_teaching_history()
</task>

<task id="4">
Create trigger function `faculty_scheduler.update_teaching_history_on_cancel()`:

```sql
-- Called on claims UPDATE to status='cancelled'
-- 1. Find the teaching_history record for this instructor/program
-- 2. Decrement block_count
-- 3. If block_count = 0, set status='cancelled', cancelled_at=NOW()
```

The function should handle the edge case where history record doesn't exist (do nothing).
</task>

<task id="5">
Create trigger `trg_claims_cancel_history` on `faculty_scheduler.claims`:
- AFTER UPDATE
- FOR EACH ROW
- WHEN (OLD.status = 'confirmed' AND NEW.status = 'cancelled')
- EXECUTE FUNCTION faculty_scheduler.update_teaching_history_on_cancel()
</task>

<task id="6">
Create function `faculty_scheduler.complete_past_teaching_history()`:

```sql
-- Called by n8n scheduled job (daily)
-- Updates all 'pending' records where COALESCE(end_date, start_date) < CURRENT_DATE
-- Sets status='completed', completed_at=NOW()
-- Returns count of updated records for logging
```

This function should be idempotent (safe to run multiple times).
</task>

<task id="7">
Create helper view `faculty_scheduler.instructor_history_summary`:

```sql
SELECT
  instructor_id,
  COUNT(*) as total_programs,
  COUNT(*) FILTER (WHERE status = 'completed') as completed_count,
  COUNT(*) FILTER (WHERE status = 'pending') as pending_count,
  COUNT(*) FILTER (WHERE status = 'cancelled') as cancelled_count,
  MAX(start_date) as last_program_date
FROM faculty_scheduler.teaching_history
GROUP BY instructor_id;
```

This view supports quick summary lookups in the dashboard.
</task>
