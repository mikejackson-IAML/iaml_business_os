# Meeting Prep Builder: Configuration Wizard

This skill guides you through questions to build the best, most comprehensive meeting prep tailored to your specific needs, meeting type, and goals.

---

## When to Use This Skill

Use this skill when:
- You have a new type of meeting that doesn't fit existing templates
- You want to customize meeting prep for a specific situation
- You're unsure what information you need before a meeting
- You want to optimize your prep process for recurring meeting types

---

## PHASE 1: MEETING TYPE IDENTIFICATION

Start by asking these questions to understand the meeting context:

### Question 1: Meeting Category

> What type of meeting is this?

| Option | Description | Follow-up Focus |
|--------|-------------|-----------------|
| **A) Sales/Discovery** | First conversation with a potential customer | Qualification, needs assessment, proposal prep |
| **B) Sales/Demo** | Showing product/service to interested prospect | Feature mapping, objection handling, closing |
| **C) Sales/Negotiation** | Discussing terms, pricing, contracts | Leverage points, walk-away points, concessions |
| **D) Partnership** | Exploring strategic partnership | Mutual benefits, alignment, deal structure |
| **E) Vendor Evaluation** | You're evaluating a vendor | Requirements, questions, red flags |
| **F) Internal Strategic** | Planning, strategy, decision-making | Background, stakeholders, decision criteria |
| **G) Client Success** | Check-in with existing client | Health indicators, upsell, retention |
| **H) Networking** | Conference, event, 1:1 networking | Common ground, mutual value, follow-up |
| **I) Media/PR** | Interview, podcast, speaking | Talking points, narrative, Q&A prep |
| **J) Custom** | Something else entirely | Fully custom configuration |

---

### Question 2: Your Role in the Meeting

> What's your role in this meeting?

| Option | Description | Prep Implications |
|--------|-------------|-------------------|
| **A) Seller/Pitcher** | You're selling or presenting something | Need to understand their needs, prepare value props |
| **B) Buyer/Evaluator** | You're evaluating something | Need to prepare questions, criteria, red flags |
| **C) Partner/Equal** | Mutual exploration | Need to understand mutual fit, shared goals |
| **D) Expert/Advisor** | Providing expertise or advice | Need to understand their situation, prepare recommendations |
| **E) Listener/Learner** | Primarily gathering information | Need to prepare questions, take notes efficiently |

---

### Question 3: Stakes Level

> How important is this meeting?

| Option | Description | Prep Depth |
|--------|-------------|------------|
| **A) Critical** | High-value deal, key relationship, career-impacting | Deep research, multiple scenarios, full brief |
| **B) Important** | Significant opportunity but not make-or-break | Solid research, key questions prepared |
| **C) Standard** | Regular meeting, moderate importance | Basic research, standard framework |
| **D) Low Stakes** | Exploratory, casual, or low priority | Quick scan, minimal prep |

---

### Question 4: Recurring or One-Time?

> Is this a one-time meeting or part of a recurring series?

| Option | Description | Prep Implications |
|--------|-------------|-------------------|
| **A) One-Time** | Single meeting | Full prep each time |
| **B) Recurring - Same Person** | Regular meetings with same person | Update-focused, relationship building |
| **C) Recurring - Different People** | Same type of meeting, different people | Template-based, customize per person |

## PHASE 2: INFORMATION NEEDS ASSESSMENT

Based on meeting type, ask about specific information needs:

### Question 5: Company/Organization Research

> Do you need information about the other party's organization?

If YES, ask:
- [ ] Company overview (size, industry, revenue)
- [ ] Recent news (funding, leadership changes, M&A)
- [ ] Competitive landscape
- [ ] Financial health
- [ ] Culture and values
- [ ] Technology stack
- [ ] Key challenges/pain points
- [ ] Open positions (signals growth areas)

### Question 6: Person/Contact Research

> Do you need information about the specific person(s) you're meeting?

If YES, ask:
- [ ] Professional background (LinkedIn)
- [ ] Current role and tenure
- [ ] Decision-making authority
- [ ] Communication style preferences
- [ ] Recent activity (posts, articles, speaking)
- [ ] Shared connections or common ground
- [ ] Personal interests (if relevant for rapport)

### Question 7: Relationship Context

> Do you have any prior relationship with this person/company?

If YES, ask:
- [ ] Previous conversations or meetings
- [ ] Past proposals or quotes
- [ ] Historical engagement with your content
- [ ] Any open issues or concerns
- [ ] Last touchpoint and outcome

### Question 8: Competitive Intelligence

> Do you need competitive intelligence for this meeting?

If YES, ask:
- [ ] Other vendors they're evaluating
- [ ] What they're currently using
- [ ] Competitor strengths/weaknesses
- [ ] Differentiation points to emphasize

|--------|---------------|

## PHASE 5: OUTPUT CONFIGURATION

### Question 16: Brief Format

> How do you want the meeting prep delivered?

- [ ] **Quick Brief** - 1-page, bullet points, <2 min read
- [ ] **Standard Brief** - 2-3 pages, key sections, <5 min read
- [ ] **Comprehensive Brief** - Full research, scenarios, 5-10 min read
- [ ] **Checklist Only** - Just questions and action items

### Question 17: Specific Sections

> Which sections should be included in your brief?

- [ ] Company Snapshot
- [ ] Contact Profile
- [ ] Prior Relationship Summary
- [ ] Industry Context
- [ ] Recent News
- [ ] Competitive Intelligence
- [ ] Questions to Ask
- [ ] Talking Points / Key Messages
- [ ] Objection Handling
- [ ] Red Flags to Watch
- [ ] Opportunities to Explore
- [ ] Next Steps Script
- [ ] Follow-Up Actions

## PHASE 7: SAVE OR USE

After configuration, offer these options:

1. **Use Now**: Generate a prep brief for a specific upcoming meeting
2. **Save as Command**: Create a new `/meeting-prep-[type]` command for reuse
3. **Save as Template**: Store the configuration for future reference
4. **Refine**: Go back and adjust any configuration options
