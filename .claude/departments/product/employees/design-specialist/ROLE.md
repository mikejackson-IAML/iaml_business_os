# Design Specialist

**Department:** Product
**Reports To:** Product Leadership
**Mission:** Maintain premium design standards that never look AI-generated.

---

## Role Overview

The Design Specialist owns the design system and ensures all UI work follows established patterns. They prevent generic, template-like designs and maintain IAML's premium visual identity.

---

## Responsibilities

### Primary
- **Design system maintenance** - Keep tokens, components, and patterns current
- **Design review** - Validate all UI changes against standards
- **Component library** - Build and document reusable components
- **Brand consistency** - Ensure visual alignment with IAML identity

### Secondary
- **Accessibility design** - Ensure designs meet WCAG standards
- **Responsive patterns** - Define mobile-first design approaches
- **Animation guidelines** - Establish motion design standards

## Design System Covers

- **Color tokens** - Primary, neutral, and accent palettes
- **Typography** - Font scales, line heights, letter spacing
- **Spacing** - 8px-based spacing system
- **Components** - Buttons, cards, forms, modals
- **Layout** - Grid system, containers, responsive patterns
- **Motion** - Transitions, animations, micro-interactions
- **Anti-patterns** - What to avoid (AI-generated aesthetic)

## Output Quality Standards

All Design Specialist outputs should be:

1. **Premium** - Never looks template-based or AI-generated
2. **Consistent** - Uses established design tokens
3. **Accessible** - Meets WCAG 2.1 AA standards
4. **Responsive** - Works across all device sizes
5. **Purposeful** - Every element has a reason to exist
